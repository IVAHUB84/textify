Пользователь запросил автоматический прогон релиза через Lead.

**Аргумент:** `$ARGUMENTS` — версия релиза `vX.Y.Z`. **Обязателен.**

## Что делать

1. Проверь аргумент:
   - Если `$ARGUMENTS` пустой или не соответствует формату `vX.Y.Z` — остановись и попроси пользователя указать версию.
2. Проверь предусловие: существует ли `releases/release-spec-vX.Y.Z.md`.
   - Если нет — остановись и предложи пользователю сначала запустить `/ba` и `/adr` для соответствующего требования.
3. Покажи пользователю короткий план: «Запускаю Lead для vX.Y.Z. Шаги: implement → arch-check → review → release → deploy (с подтверждением push'а).» и спроси подтверждение.
4. После подтверждения — запусти subagent через Agent:
   - `subagent_type`: `lead`
   - `description`: «Автоматический прогон workflow для релиза vX.Y.Z»
   - `prompt`: версия релиза vX.Y.Z, путь к спецификации, явное указание: «Выполни процесс из системного промпта. Не пиши код сам, не делай git push. Push — только через мейнтейнера и только после подтверждения пользователя.»

## Важно
Lead — это автомат, который вызывает остальных subagents. На шаге `deploy` он **обязан** остановиться перед push и запросить подтверждение пользователя. Если он этого не делает — это баг, останови и сообщи пользователю.
