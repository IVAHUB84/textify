# Textify

Telegram-бот, который **бесплатно** превращает голосовые/аудио и изображения в чистый структурированный текст.

Бот: [@YourTextifyBot](https://t.me/YourTextifyBot).

## Возможности

- **OCR изображений** — Tesseract с предобработкой (выравнивание, бинаризация, deskew) и quality gate; языки rus + eng.
- **Распознавание аудио** — Cloudflare Whisper `large-v3-turbo` как основной провайдер; локальный faster-whisper как фолбэк при любом сбое облака или при `ASR_PROVIDER=local`.
- **Структурирование текста** — Cloudflare Workers AI приводит распознанный текст к читаемой структуре; форматирование — Telegram HTML (жирные заголовки, маркированные списки).
- **Действия над результатом** — inline-кнопки под результатом: «Кратко», «Перевести» (RU↔EN), «Задачи» (action items / чек-лист). Для длинного аудио (≥ 60 с) — дополнительно «⏱ Тайм-коды» и «Субтитры» (.srt). Для изображений — «📄 PDF» (searchable PDF с текстовым слоем Tesseract).
- **Персистентный кэш результатов** — SQLite write-through рядом со статистической базой: кнопки действий работают после редеплоя и после истечения in-memory TTL.
- **Гард 20 МБ** — понятное сообщение при попытке отправить файл крупнее лимита Telegram getFile.
- **Статистика** — команда `/stats` для администратора: уникальные пользователи, всего сообщений, разбивка по типам.

Языки контента: **русский + английский** (Whisper определяет язык автоматически, OCR подключён с пакетами `rus` + `eng`, перевод — ru↔en, направление определяет LLM).

## Деплой

Бот развёрнут на **ivahub** (1 vCPU / 1.9 GB RAM, long-polling, Docker).

Образ публикуется в GHCR и деплоится через CI/CD (`push` в `main` → `.github/workflows/deploy.yml`).

Профиль поставки — «бот без HTTP-порта»: нет входящих портов, нет Traefik/edge, только `restart: unless-stopped` и volume данных статистики.

Статистика хранится в SQLite-файле на хосте: `/opt/apps/textify/data` → `/app/data` (bind-mount volume). Значение `STATS_DB_PATH` указывает на этот путь внутри контейнера.

## Особенности

- **Пиннинг Telegram-DC.** На ivahub часть IP Telegram блокируется на сетевом уровне. `docker-compose.yml` прописывает `extra_hosts` с достижимым Telegram-DC, чтобы long-polling работал.
- **Офлайн-модель Whisper.** Веса faster-whisper (`base`) вшиваются в Docker-образ на этапе сборки (`RUN python -c "from faster_whisper import WhisperModel; ..."`). В рантайме `HF_HUB_OFFLINE=1` и `TRANSFORMERS_OFFLINE=1` блокируют попытки скачивания — модель работает офлайн.
- **Гео-ограничения Groq.** Groq API недоступен с российских IP (403). Дефолт — Cloudflare Workers AI, который работает с RU-IP без прокси.

## Эксплуатация

### Режим групп — обязательный шаг (Group Privacy OFF)

Для работы проактивных офферов (кнопка «Распознать») бот должен видеть все сообщения группы. По умолчанию Telegram включает для ботов **Group Privacy ON** — бот получает только адресованные ему сообщения.

**Обязательная последовательность при первом добавлении бота в группу:**

1. Открыть [@BotFather](https://t.me/BotFather) → `/setprivacy`.
2. Выбрать бота → выбрать **Disable**.
3. **Заново добавить бота в группу** (удалить и добавить снова) — смена режима вступает в силу только для новых участников.

Что означает Privacy OFF:

- Бот видит весь трафик группы, но **реагирует только на поддерживаемое медиа** (голосовые, аудио, фото, документы): отправляет оффер-кнопку.
- На обычный текст и прочие сообщения бот **не отвечает**.
- Если шаг не выполнен (Privacy ON), бот **не падает**, но офферы не появляются; прежний триггер `/textify`/упоминание продолжает работать.

### Данные статистики

SQLite-база хранится по пути `STATS_DB_PATH` (дефолт `/app/data/stats.db`), смонтированному из `/opt/apps/textify/data` на хосте. Переживает передеплой при сохранении volume.

### Переключение провайдеров

```
ASR_PROVIDER=local         # использовать только локальный faster-whisper
ASR_PROVIDER=cloudflare    # CF Whisper + локальный фолбэк (дефолт)

LLM_PROVIDER=cloudflare    # CF Workers AI (дефолт)
LLM_PROVIDER=groq          # Groq (требует GROQ_API_KEY, не работает с RU-IP)
```

## Публичный контракт 1.0.0

С версии 1.0.0 следующий контракт стабилен. Его ломающее изменение = MAJOR.

**Команды:** `/start`, `/help`, `/stats`.

**Входные каналы:**
- фото (`message.photo`)
- изображение-документ (MIME `image/*`)
- голосовое (`message.voice`)
- аудио (`message.audio`)
- аудио-документ (MIME `audio/*`)

**Формат ответа:** Telegram HTML (`parse_mode=HTML`): жирные заголовки, маркированные списки, экранирование `< > &`; при ошибке разметки — деградация в плоский текст. Отдача через `send_result` — одно сообщение / серия по естественным границам / файл `result.txt` при очень длинном результате. При нераспознанном тексте/речи — служебное сообщение.

**Выдача результата:**
- **Личка:** сначала суть (1–2 предложения) + кнопки «Полностью», «Кратко», «Перевести», «Задачи»; для длинного аудио — дополнительно «⏱ Тайм-коды», «Субтитры»; для изображений — «📄 PDF».
- **Группы:** полный структурированный текст + кнопки «Кратко», «Перевести», «Задачи»; контекстные кнопки по правилам (аудио/изображение).

**Inline-действия:**
- «Полностью» (`act:full`) — личка: раскрывает полный структурированный текст.
- «Кратко» (`act:sum`) — выжимка 2–5 тезисов, заметно короче источника.
- «Перевести» (`act:tr`) — перевод распознанного текста RU↔EN, направление определяет LLM.
- «Задачи» (`act:task`) — action items / решения в виде чек-листа.
- «⏱ Тайм-коды» (`act:ts`) — построчные метки `[MM:SS]` для аудио ≥ 60 с.
- «Субтитры» (`act:srt`) — файл `.srt` для аудио ≥ 60 с.
- «📄 PDF» (`act:pdf`) — searchable PDF для изображений.

Источник данных для действий — кэш по `(chat_id, message_id)` (in-memory LRU+TTL + SQLite write-through). При недоступности данных — понятное служебное сообщение, без падения. При исчерпании дневного LLM-бюджета (`CF_DAILY_BUDGET`) — понятное сообщение для LLM-действий; тайм-коды/субтитры/PDF — локальные, без бюджета.
