# adrs/

Architecture Decision Records. Создаются архитектором (`/adr`).

## Формат имени
`adr-NNN.md` — сквозная нумерация, независимая от номеров requirements/ba-req.

## Содержимое
Описание контекста, рассматриваемых вариантов, выбранного решения, последствий. Один ADR — одно архитектурное решение.

Полный шаблон — в системном промпте агента `architect` (`.claude/agents/architect.md`).

## Жизненный цикл статуса
- `предложено` — обсуждается
- `принято` — действует
- `отклонено` — рассмотрено и отвергнуто
- `устарело (superseded by ADR-MMM)` — заменено другим ADR

Существующие принятые ADR не редактируются; вместо этого создаётся новый ADR со ссылкой на старый.
