# ba-req/

Каталог оформленных бизнес-требований. Создаётся бизнес-аналитиком (`/ba`).

## Формат имени
`ba-requirements-NNN.md`, где `NNN` совпадает с номером исходного требования из `requirements/`.

## Содержимое
Структурированный документ с разделами: контекст, scope, действующие лица, функциональные требования, нефункциональные требования, ограничения, критерии приёмки, открытые вопросы.

Полный шаблон — в системном промпте агента `business-analyst` (`.claude/agents/business-analyst.md`).

## Дальше
`/adr NNN vX.Y.Z` → архитектор создаст ADR и спецификацию релиза.
