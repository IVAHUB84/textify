# BA-требования 011: Гарантия бесплатности при росте (дневной бюджет Cloudflare с авто-деградацией на локальные движки) + дискаверабилити

- **Источник:** `requirements/req-textify-011.md`
- **Дата:** 2026-06-23
- **Статус:** черновик

## 1. Контекст и бизнес-цель

Textify — Telegram-бот, который **бесплатно** превращает голосовые/аудио и изображения в чистый
структурированный текст (см. `README.md`, `CLAUDE.md`). Релизы v0.1.0–v1.0.0 полностью в проде на
ivahub (1 vCPU / 1.9 GB RAM, long-polling, Docker, образ из GHCR; профиль поставки «бот без
HTTP-порта» — ADR-001). Продукт стабилизирован и зафиксирован как 1.0 (ADR-010): функциональность
закрыта, контракт стабилен.

Сверхцель продукта — **максимум пользователей бесплатно: и для пользователей, и для владельца**.
Сейчас две ключевые внешние операции — транскрипция (CF Whisper, ADR-008) и LLM-обработка
(структурирование ADR-005, действия «Кратко»/«Перевести» ADR-009) — выполняются через бесплатный
tier **Cloudflare Workers AI = 10 000 «нейронов»/день** (сброс в 00:00 UTC). Пока пользователей
немного, лимит не достигается. Но рост трафика (а в планах — режим групп v1.2.0 и виральная петля
v1.3.0) может вывести суточное потребление за free-tier, и Cloudflare начнёт **тарифицировать**
вызовы. Это прямо противоречит сверхцели «бесплатно для владельца».

Релиз v1.1.0 закрывает два связанных пробела:

- **Гарантия бесплатности при росте (главное).** Ввести персистентный **дневной счётчик** обращений
  к Cloudflare за текущие UTC-сутки и **дневной бюджет** (`CF_DAILY_BUDGET`). При достижении бюджета
  — **авто-деградация на локальные бесплатные движки без отказа пользователю**: ASR переключается на
  локальный faster-whisper (путь существует, ADR-008), LLM-структурирование — на тихий фолбэк к
  сырому тексту (существует, ADR-005), а действия «Кратко»/«Перевести» — на понятное сообщение
  «дневной бесплатный лимит исчерпан, попробуйте завтра» (локальной LLM-замены нет). Счётчик
  сбрасывается по смене UTC-даты; переход в деградацию логируется (наблюдаемость v1.0.0, ADR-010).
- **Дискаверабилити (дешёвый органический приток).** Проверить и усилить ключевые слова в уже
  заданных метаданных бота (`set_my_commands`/`set_my_description`/`set_my_short_description` —
  v1.0.0, `handlers/commands.py`) под поиск Telegram. Подготовить готовый текст листинга для
  бесплатных каталогов ботов (`docs/listing.md`); сама подача в каталоги — ручной шаг владельца,
  фиксируемый в чек-листе деплоя.

Эти две части независимы по реализации, но объединены одной целью: обеспечить **рост без затрат**
(страховка бюджета) и **органический приток** (дискаверабилити), сохранив продукт бесплатным для
всех.

Стейкхолдеры:
- **Владелец продукта / эксплуатант (заказчик).** Получает гарантию, что рост трафика **никогда** не
  приведёт к оплате Cloudflare (автоматическая страховка), и готовый материал для бесплатного
  продвижения бота.
- **Конечный пользователь Telegram.** При исчерпании дневного бюджета **продолжает** получать
  распознавание (локальные движки) без отказа; для LLM-действий получает понятное сообщение вместо
  ошибки. Дополнительно — легче находит бота через поиск Telegram и каталоги.
- **Команда разработки (последующие агенты).** Получает переиспользуемый механизм дневного учёта/
  бюджета как основу под будущий рост (группы v1.2.0, виральность v1.3.0).

## 2. Область (scope)

### В рамках задачи

- **Персистентный дневной счётчик обращений к Cloudflare.**
  - Счётчик считает обращения к Cloudflare Workers AI (CF Whisper для ASR + CF LLM для
    структурирования и действий «Кратко»/«Перевести») за **текущие UTC-сутки**.
  - Хранение — **персистентное**, переживает передеплой/перезапуск контейнера: переиспользовать уже
    существующее персистентное состояние SQLite в volume из v0.6.0 (ADR-006) и тот же каталог
    данных (`STATS_DB_PATH`/volume `/opt/apps/textify/data`).
  - Инкремент — **concurrency-safe**, по образцу `services/stats.py` (соединение-на-операцию, WAL,
    `busy_timeout`, атомарный UPSERT). Корректность при параллельных обращениях из разных потоков
    `asyncio.to_thread`.
  - **Сброс по смене UTC-даты:** счётчик ведётся в разрезе текущей UTC-даты; при наступлении новых
    UTC-суток актуальным становится новый дневной счётчик (бюджет восстанавливается автоматически
    без ручного вмешательства и без падения процесса).
- **Дневной бюджет и порог деградации.**
  - Дневной бюджет задаётся переменной окружения **`CF_DAILY_BUDGET`** с **консервативным значением
    по умолчанию** (с запасом под worst-case стоимость нейронов на вызов, чтобы гарантированно не
    выйти за free-tier 10 000 нейронов/сутки). Конкретное значение дефолта — за архитектором
    (см. ОВ-1).
  - Способ учёта «стоимости» обращения (точный подсчёт нейронов, если CF их возвращает, vs
    консервативный прокси по числу вызовов) — за архитектором; требование допускает **консервативный
    прокси по числу вызовов** (см. «Вне рамок» исходного требования и ОВ-2).
- **Авто-деградация при достижении бюджета (без отказа пользователю).**
  - **ASR:** при достижении дневного бюджета CF Whisper **не вызывается** — транскрипция выполняется
    локальным faster-whisper (путь существует, ADR-008; локальная модель `base` — в образе).
    Пользователь получает транскрипт (медленнее/качество `base`), отказа нет.
  - **LLM-структурирование:** при достижении бюджета CF LLM не вызывается — применяется существующий
    **тихий фолбэк на сырой текст** (ADR-005). Пользователь получает распознанный текст без
    структурирования, отказа нет.
  - **Действия «Кратко»/«Перевести»:** при достижении бюджета CF LLM не вызывается; локальной
    замены LLM нет — пользователю показывается **понятное русское сообщение** вида «дневной
    бесплатный лимит исчерпан, попробуйте завтра» (точная формулировка — за реализацией, стиль —
    как существующие служебные сообщения). Бот не падает.
  - Переход в режим деградации **логируется** (видимо в `docker logs`, наблюдаемость ADR-010): факт
    исчерпания бюджета и переключение на локальный/фолбэк-путь.
- **Дискаверабилити: усиление метаданных бота.**
  - Проверить и при необходимости **усилить ключевые слова** в уже заданных
    `set_my_commands`/`set_my_description`/`set_my_short_description` (`handlers/commands.py`,
    v1.0.0) под поиск Telegram: распознавание речи, транскрипция, OCR, перевод, голос в текст,
    RU/EN. Соблюсти лимиты Telegram (description ≤ 512, short_description ≤ 120 символов).
- **Дискаверабилити: текст листинга для каталогов.**
  - Подготовить документ `docs/listing.md` с готовым текстом для бесплатных каталогов ботов
    (BotoStore, TeleHunt и т. п.): название, короткое и полное описание, категории, теги/ключевые
    слова. Текст на русском (и/или с английским вариантом, если уместно для каталогов — за
    реализацией).
  - **Подача в каталоги — ручной шаг владельца**; зафиксировать его в чек-листе деплоя.

### Вне рамок (explicitly out of scope)

- **Режим групп** — следующий релиз v1.2.0 (исходное требование, `roadmap.md`).
- **Реферальная / виральная петля** — релиз v1.3.0 (исходное требование, `roadmap.md`).
- **Точный посчёт нейронов на вызов, если CF его не возвращает** — допускается консервативный прокси
  по числу вызовов (исходное требование).
- **Изменение логики распознавания, структурирования, действий и отдачи результата** по существу
  (ADR-005/007/008/009): релиз добавляет **гейт по бюджету** перед обращением к CF и переиспользует
  **уже существующие** локальные/фолбэк-пути, не меняя их поведение в штатном режиме.
- **Смена провайдеров по умолчанию** (CF для ASR/LLM) и переключатели `ASR_PROVIDER`/`LLM_PROVIDER` —
  наследие ADR-005/008, релиз их не отменяет; авто-деградация по бюджету — дополнительный поверх них
  механизм.
- **Изменение профиля поставки** (ADR-001): новых входящих портов, новых внешних зависимостей и
  новых направлений исходящего трафика не вводится; переиспользуется существующий volume данных
  (ADR-006) и существующее направление к `api.cloudflare.com` (ADR-005/008).
- **Автоматическая подача листинга в каталоги, парсинг/мониторинг каталогов, рекламные кампании** —
  релиз готовит только текст; подача ручная.
- **Кэширование/дедупликация CF-запросов, чанкинг крупного аудио, переход на webhook, апгрейд
  сервера** — вне рамок.
- **Отдельный пользовательский экран/команда «остаток дневного бюджета»** — не требуется
  (деградация прозрачна; явная индикация остатка — за рамками, если архитектор не сочтёт нужным,
  ОВ-5).

## 3. Действующие лица и сценарии использования

**Владелец продукта / эксплуатант.** Цель — рост аудитории без затрат и органический приток.

- **Сценарий A (страховка бюджета).** Трафик растёт; суточное потребление Cloudflare приближается к
  free-tier. По достижении `CF_DAILY_BUDGET` бот автоматически переключается на локальные/фолбэк-
  пути — оплата Cloudflare **не наступает**. Владельцу не нужно вмешиваться.
- **Сценарий B (диагностика деградации).** Эксплуатант видит в `docker logs` (ADR-010) запись о
  том, что дневной бюджет исчерпан и обработка идёт локально/в фолбэке — понимает причину
  возможного замедления/упрощения вывода.
- **Сценарий C (продвижение).** Владелец берёт готовый текст из `docs/listing.md` и вручную подаёт
  бота в бесплатные каталоги; шаг зафиксирован в чек-листе деплоя, поэтому не теряется.

**Конечный пользователь Telegram.** Цель — получить результат и найти бота.

- **Сценарий D (распознавание при исчерпанном бюджете).** Пользователь присылает аудио после
  исчерпания дневного бюджета: получает транскрипт от локального whisper (без отказа). Присылает
  фото: OCR локальный и так (Tesseract) — без изменений. Структурирование при исчерпанном бюджете
  тихо деградирует в сырой текст — пользователь получает распознанный текст.
- **Сценарий E (LLM-действие при исчерпанном бюджете).** Пользователь нажимает «Кратко»/«Перевести»
  после исчерпания бюджета: получает понятное сообщение «дневной бесплатный лимит исчерпан,
  попробуйте завтра» вместо ошибки/зависания; бот не падает.
- **Сценарий F (нормальный режим, бюджет не исчерпан).** Пока дневной бюджет не достигнут, всё
  работает как в v1.0.0: CF Whisper, CF-структурирование, действия — без изменений.
- **Сценарий G (новые сутки).** После 00:00 UTC дневной счётчик считается заново — облачные пути
  снова доступны без ручного вмешательства.
- **Сценарий H (поиск/каталог).** Новый пользователь находит бота через поиск Telegram (усиленные
  ключевые слова в описании) или в каталоге ботов (поданный листинг) и начинает им пользоваться.

**Команда разработки (последующие агенты).** Цель — опираться на механизм бюджета при будущем росте.

- **Сценарий I (масштабирование).** При реализации групп (v1.2.0) и виральности (v1.3.0)
  разработчик переиспользует существующий дневной учёт/бюджет CF как готовую страховку от выхода за
  free-tier.

## 4. Функциональные требования

- **ФТ-1.** Ведётся персистентный счётчик обращений к Cloudflare Workers AI за текущие UTC-сутки.
  Счётчик переживает перезапуск/передеплой контейнера, так как хранится в уже существующем
  персистентном состоянии (SQLite в volume, ADR-006; тот же каталог данных `STATS_DB_PATH`/volume).
- **ФТ-2.** Учитываются обращения ко всем платным по нейронам путям Cloudflare: ASR через CF Whisper
  (ADR-008) и LLM через CF (структурирование ADR-005, действия «Кратко»/«Перевести» ADR-009).
  Полностью локальные операции (Tesseract-OCR, локальный whisper-фолбэк) **не** учитываются (они
  бесплатны и не тратят бюджет CF).
- **ФТ-3.** Инкремент счётчика **concurrency-safe** по образцу `services/stats.py`: соединение-на-
  операцию, WAL + `busy_timeout`, атомарный UPSERT; корректен при параллельных обращениях из
  потоков `asyncio.to_thread`. Доступ к хранилищу не блокирует event loop (через `asyncio.to_thread`,
  преемственность ADR-002/003/006).
- **ФТ-4.** Счётчик ведётся в разрезе **текущей UTC-даты**; при смене UTC-суток актуальным
  становится счётчик новых суток (бюджет восстанавливается автоматически), без ручного вмешательства
  и без падения процесса. Время — UTC (`datetime.now(timezone.utc)`), согласованно с ADR-006.
- **ФТ-5.** Дневной бюджет задаётся переменной окружения **`CF_DAILY_BUDGET`** с консервативным
  значением по умолчанию (запас под worst-case стоимость нейронов, чтобы не выйти за free-tier
  10 000 нейронов/сутки). Конкретное значение дефолта и единица измерения бюджета (число вызовов как
  прокси vs нейроны) — за архитектором (ОВ-1, ОВ-2). Отсутствие/некорректное значение `CF_DAILY_BUDGET`
  не должно ронять старт бота (поведение по умолчанию при некорректном значении — за архитектором).
- **ФТ-6.** Перед каждым обращением к Cloudflare выполняется проверка дневного бюджета. Если текущий
  дневной счётчик достиг/превысил `CF_DAILY_BUDGET` — обращение к Cloudflare **не выполняется**, и
  активируется авто-деградация (ФТ-7…ФТ-9). Если бюджет не исчерпан — обращение к CF выполняется
  штатно, а счётчик инкрементируется в связи с этим обращением.
- **ФТ-7 (деградация ASR).** При исчерпанном бюджете транскрипция аудио выполняется **локальным
  faster-whisper** (существующий путь ADR-008; модель `base` в образе) вместо CF Whisper.
  Пользователь получает транскрипт; отказа в обработке нет. Локальный путь по-прежнему проходит под
  глобальным семафором тяжёлых локальных операций (ADR-010), без дедлока.
- **ФТ-8 (деградация структурирования).** При исчерпанном бюджете LLM-структурирование не вызывает
  CF — применяется существующий **тихий фолбэк на сырой текст** (ADR-005). Пользователь получает
  распознанный (нераспознанный → прежнее служебное сообщение) текст без структурирования; отказа нет.
- **ФТ-9 (деградация действий «Кратко»/«Перевести»).** При исчерпанном бюджете действия
  «Кратко»/«Перевести» не вызывают CF; локальной LLM-замены нет, поэтому пользователю показывается
  **понятное русское сообщение** о том, что дневной бесплатный лимит исчерпан и стоит попробовать
  завтра (через `callback.answer`/`message.answer`, по образцу обработки сбоев ADR-009). Бот не
  падает; «часики» callback снимаются.
- **ФТ-10.** Переход в режим деградации **логируется** (видимо в `docker logs`, ADR-010): факт
  достижения дневного бюджета и переключение конкретного пути (ASR→локальный / структурирование→
  сырой текст / действие→сообщение о лимите). Секреты в логи не пишутся (НФТ-1 ADR-010).
- **ФТ-11.** В штатном режиме (бюджет не исчерпан) поведение всех каналов **не меняется**
  относительно v1.0.0: CF Whisper для ASR, CF-структурирование, действия «Кратко»/«Перевести»,
  `send_result` (ADR-007), сообщения о пустом результате работают как раньше. Новое — только гейт
  по бюджету перед CF-вызовом и учёт обращения.
- **ФТ-12.** Авто-деградация не зависит от существующих переключателей `ASR_PROVIDER`/`LLM_PROVIDER`
  по существу: если оператор явно выбрал локальный режим (`ASR_PROVIDER=local`) — поведение прежнее
  (CF не вызывается и счётчик по ASR не растёт); гейт по бюджету применяется именно к фактическим
  обращениям к CF (точная композиция «переключатель + бюджет» — за архитектором, ОВ-3).
- **ФТ-13 (метаданные).** Ключевые слова в уже заданных
  `set_my_commands`/`set_my_description`/`set_my_short_description` (`handlers/commands.py`)
  проверены и при необходимости усилены под поиск Telegram (распознавание речи, транскрипция, OCR,
  перевод, голос в текст, RU/EN). Соблюдены лимиты Telegram: description ≤ 512 символов,
  short_description ≤ 120 символов; команды и подписи остаются корректными.
- **ФТ-14 (листинг).** Создан документ `docs/listing.md` с готовым текстом для бесплатных каталогов
  ботов: название, короткое и полное описание, категории, теги/ключевые слова. Текст пригоден для
  ручного копирования в каталог без доработки.
- **ФТ-15 (чек-лист деплоя).** Ручной шаг «подать/обновить листинг бота в бесплатных каталогах
  (по `docs/listing.md`)» зафиксирован в чек-листе деплоя (мейнтейнер, `/deploy`), чтобы он не
  терялся между релизами.

## 5. Нефункциональные требования

- **НФТ-1 (Бесплатность для владельца — главная цель).** При любом росте трафика суточное
  потребление Cloudflare не должно выходить за free-tier: по достижении `CF_DAILY_BUDGET`
  обращения к CF прекращаются до следующих UTC-суток. Бюджет консервативен (запас под worst-case),
  чтобы исключить тарификацию даже при пиковом распределении вызовов.
- **НФТ-2 (Отсутствие отказа пользователю).** Исчерпание бюджета не приводит к отказу в
  распознавании: ASR деградирует на локальный whisper, структурирование — на сырой текст, OCR
  изначально локален. Только для LLM-действий (нет локальной замены) пользователь получает понятное
  сообщение вместо результата.
- **НФТ-3 (Наблюдаемость).** Переход в деградацию виден в `docker logs` (преемственность ADR-010):
  эксплуатант может отличить «бот работает медленнее/проще из-за исчерпанного бюджета» от инцидента.
  Секреты в логи не попадают.
- **НФТ-4 (Неблокирование / отзывчивость).** Чтение/инкремент дневного счётчика и проверка бюджета
  не блокируют event loop (через `asyncio.to_thread` для SQLite, как ADR-006). Бот остаётся
  отзывчивым.
- **НФТ-5 (Надёжность учёта / отсутствие регрессий).** Сбой механизма учёта/бюджета (ошибка БД,
  отсутствие прав, блокировка) **не должен** ронять обработку и не должен случайно блокировать
  пользователя; поведение при сбое учёта определяется архитектором, но по умолчанию — безопасное
  (не приводящее к отказу пользователю; точная политика «fail-open в пользу пользователя vs
  fail-safe в пользу бюджета» — ОВ-4). Существующие тихие фолбэки (CF→локальный whisper,
  LLM→сырой текст) сохраняются.
- **НФТ-6 (Concurrency-safety).** Инкремент дневного счётчика корректен при параллельных обращениях
  (атомарный UPSERT, WAL, `busy_timeout`), по образцу `services/stats.py`; гонки не приводят к
  потере/задвоению учёта сверх допустимого, а тем более к падению.
- **НФТ-7 (Преемственность поставки).** Сохраняется профиль поставки «бот без HTTP-порта» (ADR-001):
  long-polling, без `ports:`/Traefik/`edge`, образ из GHCR, `restart: unless-stopped`,
  `extra_hosts`-пиннинг Telegram, volume данных (ADR-006). Релиз **не вводит** новых внешних
  зависимостей и новых направлений исходящего трафика; добавляется лишь env-переменная
  `CF_DAILY_BUDGET` (опциональная, с дефолтом) и переиспользуется существующий volume.
- **НФТ-8 (Локализация интерфейса).** Все новые пользовательские сообщения (об исчерпании дневного
  лимита для «Кратко»/«Перевести») — на **русском**, в стиле существующих служебных сообщений бота.
  Метаданные бота и `docs/listing.md` — на русском (с возможным английским вариантом для каталогов,
  если уместно), без нарушения лимитов Telegram.
- **НФТ-9 (Сопровождаемость / переиспользуемость).** Механизм дневного учёта/бюджета CF локализован
  (отдельный модуль/функции рядом со `services/stats.py` или в нём — деталь реализации за
  архитектором) и переиспользуем под будущий рост (группы v1.2.0, виральность v1.3.0).
- **НФТ-10 (Безопасность секретов).** Релиз не вводит новых секретов; `CF_DAILY_BUDGET` —
  несекретная конфигурация. Плейсхолдер `CF_DAILY_BUDGET` добавляется в `.env.example` и
  `deploy/.env.example`; реальные значения не коммитятся (`CLAUDE.md`, раздел «Секреты»).

## 6. Ограничения и допущения

Ограничения:
- **ОГР-1.** Free-tier Cloudflare Workers AI = 10 000 нейронов/сутки со сбросом в 00:00 UTC —
  внешнее ограничение вне контроля проекта; единица «нейрон» и стоимость конкретного вызова заданы
  Cloudflare. Отсюда консервативность `CF_DAILY_BUDGET` и допустимость прокси по числу вызовов.
- **ОГР-2.** Локальные движки бесплатны и доступны как страховка: Tesseract-OCR (полностью локально,
  ADR-004) и faster-whisper `base` (локальный фолбэк/режим, веса в образе, ADR-003/008). Локальной
  LLM-замены для структурирования/действий нет — структурирование деградирует в сырой текст,
  действия «Кратко»/«Перевести» деградируют в сообщение.
- **ОГР-3.** Профиль поставки (ADR-001) и границы «хендлер ↔ сервис», неблокирование event loop
  (ADR-002/003) сохраняются; релиз не вводит входящих портов, внешних БД/Redis, новых направлений
  трафика.
- **ОГР-4.** Персистентность переиспускает существующий volume данных (ADR-006); глобальный семафор
  тяжёлых локальных операций (ADR-010) применяется к локальному whisper-фолбэку и при деградации —
  без дедлока (локальный путь захватывает семафор ровно один раз, как зафиксировано в ADR-010).
- **ОГР-5.** Лимиты Telegram на метаданные: description ≤ 512, short_description ≤ 120 символов —
  усиление ключевых слов не должно их нарушать.
- **ОГР-6.** Подача листинга в каталоги — внешнее ручное действие владельца (каталоги модерируют
  заявки вручную); проект готовит только текст и фиксирует шаг в чек-листе.
- **ОГР-7.** Секреты — только через `.env`; реальный `.env` не коммитится; в репозиторий —
  плейсхолдеры.

Допущения:
- **ДОП-1.** Релизы v0.1.0–v1.0.0 в проде на ivahub; существующие пути — CF Whisper + локальный
  фолбэк (ADR-008), CF-структурирование + фолбэк на сырой текст (ADR-005), действия
  «Кратко»/«Перевести» (ADR-009), отдача `send_result` (ADR-007), логирование/индикатор/семафор
  (ADR-010) — работают; релиз добавляет поверх них **гейт по бюджету** и учёт, не меняя штатное
  поведение.
- **ДОП-2.** Метаданные бота уже задаются при старте (`setup_bot_profile` в `handlers/commands.py`,
  v1.0.0) — задача «проверить/усилить ключевые слова», а не «ввести метаданные с нуля».
- **ДОП-3.** Инфраструктура персистентного учёта существует (`services/stats.py`, SQLite в volume,
  WAL, UPSERT, `STATS_DB_PATH`) — дневной счётчик CF переиспользует тот же подход и тот же каталог
  данных, не вводя новое хранилище.
- **ДОП-4.** Граничные параметры (значение дефолта `CF_DAILY_BUDGET`, единица бюджета — нейроны или
  прокси по вызовам, политика при сбое учёта, точная композиция переключателей и бюджета, место
  размещения механизма) **делегированы архитектору** самим требованием; BA это не блокирует
  (ОВ-1…ОВ-5).
- **ДОП-5.** Образ обновляется и деплоится тем же CI/CD на ivahub (ADR-001); добавление
  `CF_DAILY_BUDGET` (опциональная env с дефолтом) и `docs/listing.md` не требует новых обязательных
  секретов и совместимо с текущей поставкой.
- **ДОП-6.** Релиз — **MINOR (v1.1.0)**: обратимо-совместимое добавление страховки и материалов
  продвижения; публичный контракт 1.0.0 (ADR-010) не ломается — для пользователя поведение либо
  идентично (бюджет не исчерпан), либо мягко деградирует без отказа в распознавании.

## 7. Критерии приёмки

- **КП-1.** Существует персистентный дневной счётчик обращений к Cloudflare за текущие UTC-сутки,
  переживающий перезапуск/передеплой контейнера (хранится в SQLite в volume, ADR-006). (ФТ-1)
- **КП-2.** Учитываются обращения к CF Whisper (ASR) и CF LLM (структурирование + «Кратко»/
  «Перевести»); полностью локальные операции (Tesseract-OCR, локальный whisper) счётчик CF не
  увеличивают. (ФТ-2)
- **КП-3.** Инкремент счётчика concurrency-safe (атомарный UPSERT, WAL, `busy_timeout`, по образцу
  `services/stats.py`) и не блокирует event loop; при параллельных обращениях учёт корректен и
  процесс не падает. (ФТ-3, НФТ-4, НФТ-6)
- **КП-4.** При смене UTC-суток актуальным становится новый дневной счётчик (бюджет автоматически
  восстанавливается) без ручного вмешательства и без падения. (ФТ-4)
- **КП-5.** Дневной бюджет читается из `CF_DAILY_BUDGET` с консервативным дефолтом (значение/единица
  — по решению архитектора); отсутствие/некорректное значение не роняет старт. Плейсхолдер
  `CF_DAILY_BUDGET` присутствует в `.env.example` и `deploy/.env.example`. (ФТ-5, НФТ-10)
- **КП-6.** Пока бюджет не исчерпан, поведение каналов идентично v1.0.0 (CF Whisper,
  CF-структурирование, действия, `send_result`, сообщения о пустом результате). (ФТ-11)
- **КП-7.** По достижении `CF_DAILY_BUDGET` обращение к CF не выполняется и активируется
  авто-деградация. (ФТ-6)
- **КП-8.** При исчерпанном бюджете аудио транскрибируется локальным faster-whisper без отказа;
  локальный путь корректно проходит под глобальным семафором (ADR-010) без дедлока. (ФТ-7, НФТ-2,
  ОГР-4)
- **КП-9.** При исчерпанном бюджете структурирование тихо деградирует в сырой текст (ADR-005);
  пользователь получает распознанный текст без отказа. (ФТ-8, НФТ-2)
- **КП-10.** При исчерпанном бюджете нажатие «Кратко»/«Перевести» даёт понятное русское сообщение об
  исчерпании дневного лимита; CF не вызывается; бот не падает, «часики» снимаются. (ФТ-9, НФТ-2,
  НФТ-8)
- **КП-11.** Переход в деградацию виден в `docker logs` (ADR-010), с указанием исчерпания бюджета и
  переключённого пути; секреты в логах отсутствуют. (ФТ-10, НФТ-3)
- **КП-12.** Сбой механизма учёта/бюджета не роняет обработку и не приводит к отказу пользователю в
  распознавании; существующие тихие фолбэки сохранены (точная политика fail-open/fail-safe — по
  решению архитектора). (НФТ-5)
- **КП-13.** Профиль поставки сохранён: long-polling, без `ports:`/Traefik/`edge`, образ из GHCR,
  `restart: unless-stopped`, пиннинг Telegram, volume данных; новых внешних зависимостей и
  направлений трафика нет; добавлена только опциональная env `CF_DAILY_BUDGET`. (НФТ-7, ОГР-3)
- **КП-14.** Метаданные бота (`set_my_commands`/`set_my_description`/`set_my_short_description`)
  проверены/усилены ключевыми словами под поиск Telegram (распознавание речи, транскрипция, OCR,
  перевод, голос в текст, RU/EN), с соблюдением лимитов (description ≤ 512, short_description ≤ 120).
  (ФТ-13, ОГР-5)
- **КП-15.** Создан `docs/listing.md` с названием, коротким и полным описанием, категориями и
  тегами, пригодный для ручного копирования в каталоги ботов. (ФТ-14)
- **КП-16.** Ручной шаг «подать/обновить листинг по `docs/listing.md`» зафиксирован в чек-листе
  деплоя. (ФТ-15)

## 8. Открытые вопросы

Блокирующих вопросов нет. Главный механизм (дневной учёт + бюджет + авто-деградация) опирается на
**уже существующие** компоненты: персистентный SQLite-учёт с concurrency-safe UPSERT
(`services/stats.py`, ADR-006), локальный whisper-фолбэк (ADR-008), фолбэк структурирования на
сырой текст (ADR-005), обработку сбоев действий (ADR-009), логирование (ADR-010). Граничные
параметры (значение бюджета, единица учёта, политика при сбое) **явно делегированы архитектору**
исходным требованием. Оформление документа не блокируется. Ниже — некритичные вопросы для `/adr`
или приёмки.

- **ОВ-1 (архитектор, `/adr`).** Конкретное консервативное значение по умолчанию `CF_DAILY_BUDGET`
  с учётом worst-case стоимости нейронов на вызов CF Whisper и CF LLM (чтобы суммарно не выйти за
  10 000 нейронов/сутки). По умолчанию — заведомо заниженное безопасное значение.
- **ОВ-2 (архитектор, `/adr`).** Единица бюджета: точный подсчёт нейронов (если CF возвращает их в
  ответе) vs консервативный прокси по числу вызовов. Требование допускает прокси по числу вызовов;
  если разные CF-вызовы (Whisper vs LLM) сильно различаются по стоимости — учитывать ли это весами.
- **ОВ-3 (архитектор, `/adr`).** Точная композиция «переключатель провайдера (`ASR_PROVIDER`/
  `LLM_PROVIDER`) + дневной бюджет»: где именно в оркестраторах (`services/transcribe.py`,
  `services/structure.py`, `services/llm.py`) ставится гейт по бюджету, чтобы он применялся ровно к
  фактическим обращениям к CF и не конфликтовал с ручным локальным режимом.
- **ОВ-4 (архитектор, `/adr`).** Политика при сбое самого механизма учёта/бюджета: fail-open
  (в пользу пользователя — разрешить CF при недоступности счётчика) vs fail-safe (в пользу бюджета —
  деградировать). По умолчанию — безопасное для пользователя поведение, не приводящее к отказу в
  распознавании (учёт — вспомогательный путь, как сбой `/stats`-учёта в ADR-006).
- **ОВ-5 (архитектор, `/adr`, опционально).** Нужна ли явная индикация остатка дневного бюджета
  (например, в `/stats` для администратора) — по умолчанию не требуется; деградация прозрачна и
  видна в логах.
- **ОВ-6 (реализация).** Структура хранения дневного счётчика: отдельная таблица (например,
  `cf_usage(date TEXT PRIMARY KEY, neurons/calls INTEGER)`) vs расширение существующей схемы; и где
  разместить модуль (рядом со `services/stats.py` или отдельный сервис) — деталь реализации.
- **ОВ-7 (проверяется при приёмке, не блокирует BA).** Подтверждение на проде ivahub: корректность
  сброса по UTC-дате, фактический переход в деградацию при искусственно заниженном `CF_DAILY_BUDGET`,
  видимость записи в `docker logs`, отсутствие отказа пользователю (КП-4, КП-7…КП-11).

Противоречий действующим ADR нет. Релиз **сохраняет** ADR-001 (профиль поставки «бот без
HTTP-порта»), ADR-002/003 (границы и неблокирование), ADR-006 (SQLite в volume, переиспользуется
под дневной счётчик), ADR-007 (`send_result`), ADR-010 (логирование, семафор, публичный контракт
1.0.0) и **переиспользует** существующие CF-пути и их фолбэки (ADR-005/008/009), добавляя поверх
них гейт по бюджету. Публичный контракт 1.0.0 (ADR-010) не ломается — релиз MINOR.
