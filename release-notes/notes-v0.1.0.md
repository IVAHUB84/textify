# Release Notes v0.1.0

- **Дата:** 2026-06-22
- **Тип релиза:** minor (первый релиз продукта)

## Кратко

Первый релиз Textify — минимальный работающий и деплоящийся каркас Telegram-бота без логики
распознавания. Основная ценность релиза — проверенный канал поставки на сервер ivahub и базовый
скелет, на который без реструктуризации наращиваются фичи v0.2.0–v0.4.0 (OCR, аудио, Groq).

## Что нового

### Возможности

- Точка входа `bot.py`: запуск в режиме long-polling (без входящего HTTP-порта).
- Команда `/start` — приветствие с описанием бота и явным указанием, что функции распознавания
  (аудио, изображения, структурирование) появятся в следующих релизах.
- Команда `/help` — краткая справка.
- Ответ-заглушка на любое обычное текстовое сообщение (не команду).
- Конфигурация из окружения: `BOT_TOKEN` читается из `.env` через `python-dotenv`; при незаданном
  или пустом токене процесс завершается с понятной диагностикой (`RuntimeError`), а не молча.
- Каркас каталогов `handlers/` (роутеры aiogram) и `services/` (заготовка для будущей логики)
  как полноценные Python-пакеты.

## Изменения публичного контракта

Первый релиз — предшествующего публичного контракта нет. Вводимый контракт:

- **Пользовательский интерфейс:** команды `/start`, `/help`, ответ-заглушка на текст (тексты на
  русском языке).
- **Конфигурация:** переменная окружения `BOT_TOKEN` (обязательна). `GROQ_API_KEY` и прочие ключи
  распознавания в этом релизе не читаются.
- **Поставка:** Docker-образ из GHCR + `deploy/docker-compose.yml` (профиль «бот без HTTP-порта»,
  ADR-001), деплой на сервер ivahub (138.16.226.66) через CI/CD.

Миграционный путь не требуется.

## Профиль поставки: бот без HTTP-порта (ADR-001)

Textify работает по long-polling и не имеет входящего HTTP-порта. Целевой сервер ivahub не имеет
Traefik и сети `edge`. Поэтому к данному релизу **не применяются** правила `deploy-guide.md` для
HTTP-сервисов: traefik-лейблы, `loadbalancer.server.port`, сеть `edge`, `ports:`, health-эндпоинт.

**Применяются** инварианты: `image:` из GHCR (не `build:`), `restart: unless-stopped`, секреты
только через `.env` (в репозитории — `.env.example` с плейсхолдерами), деплой через CI/CD.
Отклонение зафиксировано в `adrs/adr-001.md`.

## Frontend / e2e-тесты

Textify — Telegram-бот на long-polling, без web-frontend и без входящего HTTP-порта. Frontend-части
в проекте нет. Playwright и e2e-тесты к данному профилю не применимы и не создаются. Это
соответствует ADR-001 и профилю проекта.

## Известные ограничения

- Логика распознавания отсутствует: OCR (Tesseract/pytesseract) — в v0.2.0; аудио
  (faster-whisper) — в v0.3.0; структурирование через Groq — в v0.4.0.
- Медиа-вложения (фото, голос, аудио, документы) профильной логикой не обрабатываются; бот на
  них не реагирует.
- Ресурсы сервера ivahub ограничены (1 vCPU, 1.9 GB RAM) — выбор тяжёлых моделей распознавания
  и их ресурсный бюджет являются задачей релизов v0.2.0–v0.4.0.
- КП-8 (деплой на ivahub и реальный ответ бота на `/start`) проверяется только на этапе `/deploy`
  при наличии секретов и доступа к серверу.
