# Release Notes v0.2.0

- **Дата:** 2026-06-22
- **Тип релиза:** minor

## Кратко

Релиз добавляет первую реальную пользовательскую функцию — распознавание текста с изображений (OCR).
Пользователь присылает фото или изображение-файл, бот возвращает сырой распознанный текст.
Каркас v0.1.0 превращается в продукт, выполняющий полезную работу.

## Что нового

### Возможности

- Приём фото (`message.photo`) и изображений-документов (`message.document` с image-MIME) с
  распознаванием текста через Tesseract (`pytesseract`), языки `rus`+`eng` одновременно.
- При непустом результате OCR пользователь получает сырой распознанный текст.
- При пустом результате (текст не найден) — понятное русскоязычное сообщение «Текст на изображении
  не распознан.».

### Улучшения

- Новый сервисный слой `services/ocr.py`: асинхронная обёртка над `pytesseract`, синхронный вызов
  Tesseract вынесен в пул потоков через `asyncio.to_thread` — event loop остаётся отзывчивым.
- Новый хендлер `handlers/image.py` зарегистрирован до текстовой заглушки, что исключает перехват
  изображений текстовым роутером.
- `Dockerfile` обновлён: в runtime-стейдж добавлена установка `tesseract-ocr`, `tesseract-ocr-rus`,
  `tesseract-ocr-eng` с чисткой кэша apt в том же слое.

### Исправления

Нет (первая функциональная возможность, не исправление).

## Изменения публичного контракта

Обратимо-совместимое изменение (minor):

- Бот теперь обрабатывает фото и изображения-документы — возвращает распознанный текст. Ранее
  на изображение приходил ответ текстовой заглушки.
- Команды `/start`, `/help`, текстовая заглушка на обычный текст и конфигурация (`BOT_TOKEN`)
  без изменений.
- Новых переменных окружения нет. `deploy/docker-compose.yml`, `deploy/.env.example`,
  `.github/workflows/deploy.yml` не изменялись. Миграционный путь не требуется.

## Известные ограничения

- Качество распознавания определяется возможностями Tesseract и качеством присланного изображения;
  предобработка изображений вне рамок (запланирована на v0.5.0).
- При очень длинном распознанном тексте (свыше лимита Telegram) поведение не специфицировано;
  разбивка на части — v0.5.0 (ОВ-1).
- При нескольких одновременных тяжёлых изображениях время ответа растёт из-за конкуренции за 1 vCPU
  на ivahub; ограничение параллелизма — v0.5.0.
- Поддерживаются только `rus`+`eng`; выбор языка пользователем — вне рамок.
