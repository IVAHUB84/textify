# Release notes — v0.5.2

- **Дата:** 2026-06-22
- **Тип:** patch (багфикс, публичный контракт не меняется)

## Что исправлено

Бот **пытался «распознать» картинки с иконками** (сердечки, значок Wi-Fi, батарея и т.п.)
и выдавал наборы мусорных символов. Иконки — это не текст, но Tesseract всё равно пробует их
прочитать; раньше любая непустая строка OCR уходила дальше и оформлялась LLM как «текст».

Исправление в `services/ocr.py` — гейт качества по уверенности распознавания:

- вместо `image_to_string` используется один проход `image_to_data` на кандидата (текст +
  пословный confidence за один вызов — без лишней нагрузки на CPU);
- учитываются только слова с уверенностью `conf ≥ 50` и хотя бы двумя буквенно-цифровыми
  символами (одиночные символы — типичный артефакт иконок);
- если уверенного текста почти нет (порог `< 3` буквенно-цифровых символов) — возвращается
  пустая строка, и пользователь получает обычное «текст не распознан», а не мусор;
- логика «best-of-both» (предобработка vs исходное изображение) и настройки
  `--oem 1 --psm 3`, языки `rus+eng` сохранены.

Откалибровано на реальном Tesseract: изображение с текстом распознаётся как прежде,
картинка из фигур/иконок даёт пустой результат.

## Совместимость

Контракт `recognize_text` не изменился (та же сигнатура, та же асинхронность). Меняется лишь
поведение на «нетекстовых» картинках: вместо мусора — сообщение о нераспознанном тексте.
Обновление — штатным пересозданием контейнера из GHCR.

## Тесты

`pytest`: 78 passed, 3 skipped (real-Tesseract тесты пропускаются без бинаря). Добавлены тесты
гейта (низкая уверенность и символы иконок → пусто); тесты best-of-both переведены на
`image_to_data`. `ruff` чисто.
