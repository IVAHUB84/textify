# Release Notes v0.9.0

- **Дата:** 2026-06-23
- **Тип релиза:** minor

## Кратко

Под одно-сообщенческим текстовым результатом OCR и аудио-транскрипции теперь появляются inline-кнопки **«Кратко»** и **«Перевести»**. По нажатию бот через тот же Cloudflare Workers AI формирует саммари в 3–5 пунктов или перевод (ru↔en) и присылает его отдельным сообщением. Профиль поставки, зависимости и схема БД не меняются.

## Что нового

### Возможности

- Inline-кнопки «Кратко» и «Перевести» под одно-сообщенческим результатом распознавания (OCR: фото и изображение-документ; аудио: голосовое, аудио-файл, аудио-документ).
- **Саммари** (`act:sum`): структурирует смысл исходного текста в 3–5 пунктов на языке текста, не меняя язык и не добавляя факты.
- **Перевод** (`act:tr`): переводит текст с авто-определением направления по эвристике — наличие кириллицы → целевой английский, иначе → русский. Без сторонних библиотек определения языка.

### Улучшения

- `services/reply.py` — `send_result` получил опциональный параметр `reply_markup: InlineKeyboardMarkup | None = None`. Markup прикрепляется только в одно-сообщенческом режиме; при серии сообщений и файле `.txt` — игнорируется. Обратимо-совместимо: все существующие вызовы без `reply_markup` работают как прежде.
- `services/structure.py` — публичные псевдонимы `MAX_INPUT_CHARS` и `build_provider` для переиспользования в `services/llm.py` без дублирования кода провайдера.

### Новые модули

- `services/llm.py` — `summarize(text) -> str | None`, `translate(text) -> str | None`; переиспользует провайдерный слой из `structure.py`; усекает вход до 8000 символов; единичная попытка с таймаутом 30 с; любой сбой → `None` + лог, без проброса исключения.
- `handlers/actions.py` — `actions_router` с двумя `CallbackQuery`-хендлерами (`act:sum`, `act:tr`) и фабрикой клавиатуры `actions_keyboard()`.

## Изменения публичного контракта

- **Telegram-UX:** под одно-сообщенческим результатом OCR/аудио появляются inline-кнопки «Кратко» и «Перевести». Поведение при длинном результате (серия/файл), при пустом результате и логика распознавания/структурирования — без изменений.
- **Внутренний контракт `send_result`:** добавлен опциональный параметр `reply_markup` с дефолтом `None` — обратимо-совместимо.
- **Схема БД / env / порты / зависимости:** без изменений. Миграций нет. Callback'и не учитываются в статистике (`StatsMiddleware` остаётся только на `dp.message`).

## Известные ограничения

- Кнопки не показываются под длинным результатом (серия сообщений / файл `.txt`), так как `callback.message.text` содержит только часть исходного текста. Это осознанное упрощение согласно ADR-009: отдельного хранилища текстов нет.
- Эвристика направления перевода (наличие кириллицы) может ошибаться на смешанном ru/en тексте. Приемлемо для основных языков контента проекта (ru + en).
- Качество саммари/перевода зависит от модели CF и качества входного текста; абсолютная точность не гарантируется.
