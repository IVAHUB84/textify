# Release Notes v1.10.0

- **Дата:** 2026-06-25
- **Тип релиза:** minor

## Кратко

Релиз вводит **changelog-рассылку анонсов**: при выходе новой версии бот, с подтверждения
администратора, рассылает всем активным пользователям короткое «что нового» прямо в личный чат.
Поддерживаются отписка от анонсов и повторная подписка. Основная функция бота (медиа, лимиты,
gate, рефералы) не затрагивается.

## Что нового

### Возможности

- **Анонс новых версий пользователям.** При запуске бот сравнивает текущую версию с последней
  анонсированной (persistent). Если версия новая и для неё есть текст — администратор получает
  превью с кнопками «Разослать» / «Пропустить».
- **Подтверждение администратором.** Рассылка стартует только по явному нажатию «Разослать»;
  «Пропустить» помечает версию анонсированной без рассылки.
- **Фоновая троттлящаяся рассылка.** Отправка идёт в отдельной `asyncio.Task` (~20 msg/s)
  с backoff на `RetryAfter` и пропуском заблокировавших бот (`TelegramForbiddenError`).
- **Отчёт администратору** по завершении рассылки: отправлено N, пропущено M, ошибок K.
- **Отписка и подписка.** Под каждым анонсом — кнопка «🔕 Не показывать анонсы» (callback
  `ann:off`). Команды `/announces_off` и `/announces_on` доступны в любой момент.
- **Единый источник версии.** Введён `version.py` с `__version__ = "1.10.0"` и числовым
  `parse_version()` для корректного semver-сравнения (`1.10.0 > 1.9.1`).
- **Версия в `/stats`.** Команда `/stats` (только администратор) теперь показывает
  `Версия: 1.10.0`.

### Улучшения

- Идемпотентная миграция БД: новая колонка `user_stats.announcements_optout` и таблица
  `announce_state` добавляются при старте автоматически, старые данные не затрагиваются.
- Env-флаг `ANNOUNCEMENTS_ENABLED` (дефолт `true`) позволяет отключить механизм в dev/CI.

### Исправления

Нет.

## Изменения публичного контракта

- **Новые команды:** `/announces_off`, `/announces_on`.
- **Изменены:** `/stats` (admin) — добавлена строка `Версия: X.Y.Z`. `/start`, `/help` —
  без изменений.
- **Новый env:** `ANNOUNCEMENTS_ENABLED` (дефолт `true`; `false/0/no` отключает).
- **Миграция БД:** аддитивная, выполняется в коде при старте. Новая таблица `announce_state`
  (`CREATE TABLE IF NOT EXISTS`) и колонка `user_stats.announcements_optout INTEGER DEFAULT 0`
  (через `PRAGMA table_info` + `ALTER TABLE ADD COLUMN` — идемпотентно). Ручных шагов нет.
- **Новые зависимости:** нет.

## Известные ограничения

- При рестарте бота в середине рассылки повторная отправка той же версии **не производится**
  (версия помечается анонсированной до начала цикла отправки). Пользователи, не успевшие
  получить анонс до рестарта, сообщение не получат — это осознанный выбор (ADR-020 Решение 5A).
- Анонсы рассылаются только в личные чаты (`user_stats`); группы не охватываются.
- Тексты анонсов только на русском.
