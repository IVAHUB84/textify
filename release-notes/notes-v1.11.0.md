# Release Notes v1.11.0

- **Дата:** 2026-06-25
- **Тип релиза:** minor

## Кратко

Textify теперь распознаёт речь из видео: бот принимает обычные ролики (`video`), кружочки
(`video_note`) и видео, присланные файлом (`document video/*`), извлекает аудиодорожку через
PyAV и подаёт её в существующий whisper-пайплайн. Ответ — идентичен голосовому: прогрессивное
раскрытие (суть → кратко → полностью) и те же кнопки действий.

## Что нового

### Возможности

- **Новый входной канал — видео → текст (речь).** Бот принимает три формы видео:
  `video` (обычный mp4-ролик), `video_note` (кружочки), документ с mime `video/*`.
  Распознаётся именно **аудиодорожка** — не OCR кадров/субтитров.
- **Извлечение аудио через PyAV** (`services/media.py::extract_audio`): демуксирует аудиопоток
  из видео-контейнера и перекодирует в WAV PCM 16 kHz mono; работает без системного ffmpeg.
  Синхронная перекодировка выполняется в `asyncio.to_thread` под `HEAVY_LOCAL_SEMAPHORE` — event
  loop не блокируется, нагрузка на 1 vCPU предсказуема.
- **Видео без аудиодорожки** → дружелюбное русское сообщение «В этом видео нет звуковой дорожки
  — распознавать нечего.», бот не падает.
- **Ошибка декода** → понятное сообщение пользователю, обработка других сообщений не прерывается.

### Переиспользование без изменений

- Гард 20 МБ (`handlers/limits.py`) — применяется к видео как к прочему медиа.
- Дневной лимит, subscription-gate (v1.7.0) и реферальные бонусы (v1.8.0) — видео считается
  за единицу распознавания, контракт не меняется.
- `result_cache`, кнопки-действия (Кратко, Перевести, Задачи, Тайм-коды, Субтитры) — через
  `process_audio` без дублирования логики.

## Изменения публичного контракта

Раздел «Входные каналы» публичного контракта (ADR-010) дополнен обратимо-совместимо:
- `message.video` — видео-ролик;
- `message.video_note` — видео-кружок;
- `document` с `mime_type`, начинающимся на `video/` — видео-файл.

Формат ответа, набор кнопок, лимит/gate, рефералы, OCR-канал — **без изменений**.

**Видео в группах** — не входит в v1.11.0 (только личный чат; отложено, ADR-021 Решение 3A).
**Новых env / внешних зависимостей / системного ffmpeg / миграций** — нет.
**Артефакты поставки** (`Dockerfile`, `deploy/*`, `.github/workflows/deploy.yml`) — без правок.

## Известные ограничения

- Видео в групповых чатах не распознаётся — запланировано на следующий MINOR.
- Чанкинг длинного аудио из видео не вводится — наследуемый техдолг (ADR-010/ADR-021).
- Форматы, для декода которых PyAV требует дополнительных системных кодеков, не поддерживаются.
- Максимальный размер входного видео ограничен существующим гардом 20 МБ.
