# Release Notes v1.4.0

- **Дата:** 2026-06-23
- **Тип релиза:** minor

## Кратко

Textify v1.4.0 делает режим групп проактивным: бот сам замечает появившееся медиа и предлагает inline-кнопку «Распознать голос» или «Распознать текст» reply на это медиа. Расшифровка запускается только по тапу — без спама и без лишней нагрузки на сервер. Прежний групповой триггер v1.2.0 (reply + `/textify`/упоминание) сохранён как дополнительный путь. Контракт 1.0.0 лички не затронут — релиз MINOR.

## Что нового

### Возможности

- **Проактивный оффер-кнопка на медиа в группе.** При появлении поддерживаемого медиа (`voice`, `audio`, `photo`, `document image/*`, `document audio/*`) в группе/супергруппе бот отвечает reply на это сообщение одной кнопкой: «Распознать голос» для аудио/голоса и «Распознать текст» для изображений. Распознанный текст в момент оффера не отправляется.

- **Распознавание по тапу (callback `grec`).** Нажатие «Распознать» извлекает исходное медиа из `callback.message.reply_to_message`, прогоняет через стандартный пайплайн (`process_audio` / `process_photo` / `process_image_document`) и постит результат reply на исходное медиа-сообщение с кнопками «Кратко»/«Перевести». ASR в группах по-прежнему форсируется локальным faster-whisper при `GROUP_ASR_LOCAL=true` — CF-бюджет лички не расходуется.

- **Graceful-обработка граничных случаев при тапе.** Если к моменту нажатия `reply_to_message` недоступно (удалено, `InaccessibleMessage`) или не содержит поддерживаемого медиа — бот показывает русское служебное сообщение через `callback.answer(show_alert=True)`, пайплайн не вызывается, бот не падает. `callback.answer()` вызывается во всех ветках.

### Улучшения

- **Реакция только на медиа при отключённом Group Privacy.** При privacy OFF бот получает весь трафик группы, но оффер выдаёт исключительно на поддерживаемое медиа. На обычный текст и прочие сообщения — молчание.

- **Сохранён прежний триггер v1.2.0.** Путь «reply на медиа + `/textify`/упоминание» продолжает работать как дополнительный. Оба пути ведут в общий `_dispatch_media` без дублирования пайплайна.

### Исправления

- Нет багфиксов. Все изменения — новая функциональность поверх существующей инфраструктуры.

## Изменения публичного контракта

- **Личка (контракт 1.0.0, ADR-010):** без изменений. Медиа распознаётся автоматически, проактивные офферы в личке не появляются.
- **Группы:** аддитивное расширение — добавляется проактивный оффер-кнопка и распознавание по тапу (callback `grec`). Прежний триггер v1.2.0 сохранён. Изменение обратимо-совместимо.
- **Эксплуатация — новое обязательное условие:** Group Privacy OFF. Необходимо выполнить ручной шаг в @BotFather (`/setprivacy` → Disable) и заново добавить бота в группу. Без этого шага проактивные офферы не появляются; при privacy ON бот не падает, прежний триггер v1.2.0 продолжает работать.
- **Env / секреты / порты / зависимости:** без изменений. Переиспользуется существующий `GROUP_ASR_LOCAL`. Новой env не вводится. Профиль поставки ADR-001 сохранён.
- **Схема БД (`user_stats`):** без изменений. callback `grec` в статистике не считается (преемственность ADR-009).

## Известные ограничения

- Авто-распознавание медиа без тапа не введено (вне scope — риск спама и перегрузки 1 vCPU).
- Настройки на уровне группы (включение/отключение оффера, белые/чёрные списки чатов) — отложено.
- Rate-limit офферов сверх «один оффер на одно медиа» — отложено.
- Нажатие callback «grec» не учитывается в `/stats` (метрика конверсии оффер→тап — отдельным релизом).
- Без выполненного ручного шага Group Privacy OFF офферы не появляются; это ожидаемое поведение, не ошибка.

## E2E / Playwright

Textify — Telegram-бот без web UI. E2E-тесты на Playwright неприменимы. Release gate в `.github/workflows/` для e2e не создаётся.
