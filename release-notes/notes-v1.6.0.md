# Release Notes v1.6.0

- **Дата:** 2026-06-24
- **Тип релиза:** minor

## Кратко

Textify v1.6.0 закрывает три разрыва с конкурентами: Telegram-HTML вместо сырого Markdown делает выдачу читаемой; расширенное меню действий возвращает «Перевести», добавляет «Задачи», тайм-коды, субтитры и searchable PDF для фото; персистентный SQLite-кэш сохраняет кнопки рабочими после редеплоя и истечения TTL. Дополнительно: честное «Кратко» (всегда короче источника), понятное сообщение для файлов > 20 МБ и скрытая кнопка Menu.

## Что нового

### Возможности

- **Telegram HTML-форматирование.** Структурированный текст отправляется с `parse_mode=HTML`: заголовки — `<b>`, маркированные списки — `•`, спецсимволы `< > &` экранируются. При ошибке разметки Telegram — деградация в плоский текст без падения обработчика. Промпт структурирования переучен на Telegram-формат (`**bold**`, списки `- `).

- **Возврат «Перевести»** (реверс ADR-015). Функция `translate(RU↔EN)` восстановлена в `services/llm.py`; направление определяет LLM по содержимому текста. Кнопка `act:tr` доступна в личке и группах.

- **Новое действие «Задачи»** (`act:task`). `extract_tasks` выделяет action items / решения из текста в формате чек-листа. Тот же бюджет-гейт, что у остальных LLM-операций.

- **Тайм-коды и субтитры из сегментов whisper.** Для аудио ≥ 60 с под результатом появляются кнопки «⏱ Тайм-коды» (`act:ts`) и «Субтитры» (`act:srt`). Тайм-коды — построчные метки `[MM:SS]`; субтитры — документ `.srt` в стандартном формате. Данные берутся из кэша сегментов — без дополнительного LLM-вызова.

- **Searchable PDF для изображений** (`act:pdf`). Кнопка «📄 PDF» появляется под результатом фото/изображения-документа. PDF = исходное изображение + невидимый текстовый слой Tesseract `rus+eng`; текст можно выделять и искать. Без новых зависимостей (Tesseract уже установлен).

- **Персистентный кэш результатов.** `services/result_cache.py` получил SQLite-бэкенд (write-through + load-on-start). Файл `results.db` создаётся рядом с `stats.db` на старте бота. Кнопки действий работают после редеплоя и после истечения in-memory TTL. Без инициализации — чистый in-memory (поведение v1.5.0 сохранено). Байты изображений хранятся только в памяти (50 элементов max).

- **Гард 20 МБ.** Единая точка проверки `is_oversized` (`handlers/limits.py`) во всех входных хендлерах. Файл > 20 МБ → понятное сообщение до попытки скачивания; бот не молчит.

### Улучшения

- **Честное «Кратко».** Промпт `summarize` полностью переписан: few-shot на сжатие гарантирует выжимку из 2–5 тезисов без деталей и повторов, всегда заметно короче исходного текста.

- **Единая фабрика клавиатуры.** `actions_keyboard(progressive, with_extras, with_pdf)` — флаговая модель; личка и группы используют одну функцию с явными параметрами.

- **Скрыта кнопка Menu.** `setup_bot_profile` теперь вызывает `delete_my_commands` — Telegram не показывает кнопку Menu у поля ввода. Команды `/start`, `/help`, `/stats` по-прежнему работают при ручном вводе.

- **Подсказки fallback-превью** указывают на кнопки обобщённо, без перечисления конкретных названий.

### Исправления

- Деградация при пустом кэше для каждого действия — `callback.answer(show_alert=True)` без падения.
- Деградация «Перевести»/«Задачи»/«Кратко» при `BUDGET_EXCEEDED` — понятное сообщение.
- LLM-операции: `None`-ответ → «Не удалось выполнить действие. Попробуйте позже.»

## Изменения публичного контракта

Контракт 1.0.0 (ADR-010), разделы «Формат ответа» и «Inline-действия»:

- **Формат немедленной выдачи:** Telegram HTML (`parse_mode=HTML`) вместо plain text. Деградация при ошибке разметки гарантирована. Это **совместимое** изменение представления (MINOR).
- **Inline-действия (личка):** «Полностью» (`act:full`), «Кратко» (`act:sum`), «Перевести» (`act:tr`), «Задачи» (`act:task`); аудио ≥ 60 с — дополнительно «⏱ Тайм-коды» (`act:ts`) / «Субтитры» (`act:srt`); изображения — «📄 PDF» (`act:pdf`).
- **Inline-действия (группы):** «Кратко» / «Перевести» / «Задачи» + контекстные по правилам.
- **«Перевести» возвращена** — реверс ADR-015, Решение 6. ADR-015 помечен как частично superseded ADR-016.
- **Источник данных для кнопок:** кэш `(chat_id, message_id)` — in-memory + SQLite write-through; недоступность → служебное сообщение.
- **Кнопка Menu** не отображается (`delete_my_commands`).

Входные каналы, команды (`/start`/`/help`/`/stats`), логика `send_result`, профиль поставки — **не меняются**. Миграция БД не требуется: `results.db` создаётся автоматически на старте.

## Известные ограничения

- Байты изображений не персистируются — in-memory LRU (50 элементов, TTL 1 ч). Кнопка «📄 PDF» после редеплоя даёт «Изображение недоступно (устарело)».
- SQLite write-through выполняется синхронно в event loop (WAL + busy_timeout); при высокой частоте запросов может добавить задержку — на ivahub (1 vCPU / 2 GB, малый трафик) некритично.
- «Перевести»/«Задачи» в группах — охват шире, чем подразумевал исходный BA (ОВ-1): зафиксировано как технический долг в ADR-016, до подтверждения владельца продукта действует фактическое поведение.
- Mypy 2 ошибки в `services/preprocess.py` (PIL-специфичная, существует с v0.4.0) — к v1.6.0 не относятся.

## E2E / Playwright

Textify — Telegram-бот без web UI. E2E-тесты на Playwright неприменимы. Release gate в `.github/workflows/` для e2e не создаётся.
