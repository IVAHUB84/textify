# Release Notes v1.9.0

- **Дата:** 2026-06-25
- **Тип релиза:** minor

## Кратко

Кнопка «📄 PDF» теперь отдаёт аккуратную страницу, очищенную прямо перед вшиванием: перекос исправлен, тени и цветной фон стола сняты CLAHE-контрастированием. Добавлен второй режим «📄 Скан (ч/б)» с бинаризованной страницей. При любом сбое обработки бот автоматически деградирует к поведению v1.6.0 — рабочий PDF гарантирован всегда.

## Что нового

### Возможности

- **Рендер видимой страницы PDF из очищенного изображения.** К каждому фото перед вшиванием применяется пайплайн: EXIF-поворот → ресайз до 2000 px → CLAHE-контрастирование (clipLimit 2.0) → deskew. Результат — выровненная серая страница «как из сканера», а не сырое фото на столе.
- **Режим «📄 Скан (ч/б)».** Новая кнопка `act:pdfbw` рядом с «📄 PDF». Страница бинаризуется адаптивным порогом — классический вид чёрно-белого скана, компактнее по размеру.
- **Два режима одним тапом.** «📄 PDF» → режим «Документ» (серая контрастная страница, дефолт). «📄 Скан (ч/б)» → режим «Скан» (бинаризованная страница). Промежуточного меню нет.

### Улучшения

- **Согласованность текстового и видимого слоя PDF.** Tesseract строит оба слоя из одного очищенного изображения — координаты невидимого текста точно совпадают с видимой страницей, выделение текста попадает в слова.
- **Меньший размер PDF.** Страница вшивается как JPEG (quality 75), а не сырой PNG/JPEG оригинала → итоговый PDF меньше.
- **Рефакторинг `preprocess.py`.** `_deskew` разделён на `_skew_angle` и `_rotate`; добавлены хелперы `_load_oriented` и `_resize_to_target`; добавлена публичная функция `render_page(image_bytes, mode)`. Ветка OCR-текста (`preprocess_image`) не изменена.
- **Порог поворота.** При угле < 0,5° поворот пропускается — нет ложного перекоса на уже ровных документах.

### Исправления

- Нет (релиз — улучшение существующей функции).

## Изменения публичного контракта

- **Кнопка «📄 PDF» (`act:pdf`):** поведение сохраняется — searchable PDF с выделяемым текстом; видимая страница теперь очищена (выровнена, контраст/тени, JPEG, меньший размер). Деградация гарантирует: при сбое — PDF по оригиналу (v1.6.0), результат не хуже.
- **Новая кнопка «📄 Скан (ч/б)» (`act:pdfbw`):** бинаризованная страница. Добавление, не замена.
- **OCR-текст, дневной лимит, остальные кнопки, профиль поставки** — не изменяются.
- **Новые env-переменные:** нет. **Новые зависимости:** нет. **Миграция:** не требуется.

## Известные ограничения

- Режим «Документ» возвращает **серую** (не полноцветную) страницу — CLAHE по grayscale убирает цветной фон стола; полноцветный вывод — возможное будущее уточнение.
- Фиксированные константы размера/качества (2000 px, JPEG 75) — один размер для всех входов.
- Многостраничный PDF из альбома фотографий — следующее отдельное требование (вне scope v1.9.0).

**Тесты:** `pytest` — 565 passed, 3 skipped (пропущены тесты, требующие бинаря Tesseract). 3 пропуска — pre-existing, не связаны с релизом.

**Frontend / Playwright:** не применимо — проект является Telegram-ботом на Python без frontend-части.

**Критерии приёмки (КП-1…КП-11):**

- КП-1 (выровненная/контрастная страница в дефолтном режиме) — подтверждён через `test_recognize_pdf_calls_render_page_doc_mode` и `test_render_page_doc_*`: `render_page(b, "doc")` возвращает не-бинаризованное PIL-изображение, `_run_ocr_pdf` передаёт в Tesseract JPEG-перекодированный результат.
- КП-2 (searchable слой согласован с видимой страницей) — подтверждён архитектурно: один проход Tesseract по очищенному изображению (ADR-019, Решение 1A); тест `test_recognize_pdf_page_is_jpeg_encoded` фиксирует, что в Tesseract уходит ровно одна страница.
- КП-3 (режим «Скан ч/б» бинаризует) — подтверждён: `test_render_page_scan_binarized` проверяет, что `render_page(b, "scan")` возвращает изображение со значениями пикселей только из `{0, 255}`; `test_handle_pdf_bw_cache_hit_sends_document` — `handle_pdf_bw` вызывает `recognize_pdf` с `mode="scan"`.
- КП-4 (PDF не больше v1.6.0) — подтверждён через мок: JPEG quality=75, ресайз до 2000 px, `test_recognize_pdf_page_is_jpeg_encoded` фиксирует формат JPEG.
- КП-5 (деградация к оригиналу при сбое) — подтверждён: `test_recognize_pdf_render_failure_falls_back_to_original` — при исключении в `render_page` возвращается PDF по оригиналу без проброса исключения.
- КП-6 (результат не хуже v1.6.0) — подтверждён: деградация гарантирует минимум — оригинальный PDF (поведение v1.6.0) на любом входе.
- КП-7 (ровный документ не ухудшается) — подтверждён: `test_render_page_skips_rotation_below_threshold` — при угле < `_MIN_DESKEW_ANGLE` `_rotate` не вызывается; CLAHE с `clipLimit=2.0` идемпотентен на уже контрастном входе.
- КП-8 (обработка под `HEAVY_LOCAL_SEMAPHORE` вне event loop) — подтверждён кодом: `recognize_pdf` вызывает `_run_ocr_pdf` через `asyncio.to_thread` под `HEAVY_LOCAL_SEMAPHORE`; новой параллельной CV-нагрузки не вводится.
- КП-9 (прочие кнопки и лимит не изменены) — подтверждён: `recognize_text`/`_run_ocr` не тронуты; прочие callback-хендлеры не изменены; `test_actions_keyboard_extras_buttons_optional` и прочие keyboard-тесты проходят.
- КП-10 (профиль поставки не меняется) — подтверждён: нет новых env, зависимостей, артефактов поставки; `Dockerfile`/`deploy/`/`.env.example`/`.github/workflows/deploy.yml` не изменялись.
- КП-11 (подписи на русском) — подтверждён кодом: кнопка `"📄 Скан (ч/б)"`, сообщения в `_handle_pdf` на русском, в стиле существующего меню.
