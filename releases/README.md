# releases/

Спецификации релизов. Создаются архитектором (`/adr`), используются разработчиком (`/implement`), тестировщиком (`/release`) и мейнтейнером (`/deploy`).

## Формат имени
`release-spec-vX.Y.Z.md`, где `X.Y.Z` — версия по semver.

## Содержимое
Объём релиза, конкретные задачи с критериями готовности, ссылки на BA-требования и ADR, изменения публичного контракта, обязательные тесты, риски, чек-лист готовности.

Полный шаблон — в системном промпте агента `architect`.

## Принцип
Спецификация — это **исполняемая инструкция для разработчика**. Если её можно прочитать и не понять, что именно делать с кодом — это плохая спецификация.
