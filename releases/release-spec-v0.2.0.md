# Спецификация релиза v0.2.0

- **Дата планирования:** 2026-06-22
- **Тип релиза:** minor (обратимо-совместимое добавление функции OCR; публичный контракт v0.1.0 не ломается)
- **Цели:** превратить каркас-бот в продукт с первой реальной функцией — распознавание текста
  с изображений (OCR, `rus`+`eng`): пользователь присылает фото или изображение-документ, бот
  возвращает сырой распознанный текст.

## Источники
- BA-требования: `ba-req/ba-requirements-002.md`
- ADR: `adrs/adr-002.md` (база — `adrs/adr-001.md`)

## Объём релиза

Минимальность обязательна (`CLAUDE.md`, принцип «Минимальные изменения»). В рамках — только OCR.
**Вне рамок** (НЕ реализовывать): структурирование через Groq (v0.4.0), аудио/faster-whisper
(v0.3.0), лимиты размера файла, индикация прогресса, разбивка длинного текста, расширенная
обработка ошибок и предобработка изображений (v0.5.0), выбор языка пользователем, многостраничные
файлы/PDF, хранение/аналитика/БД. Зависимости аудио (faster-whisper, ffmpeg) и структурирования
(Groq SDK, `GROQ_API_KEY`) не вводятся.

### Задача 1: Сервис OCR (`services/ocr.py`)
- **Что:** асинхронная обёртка над `pytesseract`, распознающая текст на изображении с языками
  `rus`+`eng`. Синхронный вызов Tesseract выполняется вне event loop (через `asyncio.to_thread`
  или executor) — см. ADR-002, Решение 1. Возвращает строку распознанного текста (для пустого
  результата — пустая/«пробельная» строка, решение об ответе принимает хендлер).
- **Где (файлы/модули):** `services/ocr.py`.
- **Контракт:** публичная асинхронная функция, принимающая байты изображения (или
  `PIL.Image`/буфер — на усмотрение разработчика, но не путь к файлу на диске и не объект aiogram)
  и возвращающая `str`. Жёстко зафиксировано: `lang="rus+eng"`. Модуль не импортирует aiogram и не
  обращается к Telegram — только к `pytesseract`/`Pillow`.
- **Критерии готовности:** функция асинхронная; распознаёт текст на тестовом изображении с
  `rus`+`eng`; блокирующий вызов Tesseract не исполняется напрямую в корутине (вынесен в поток);
  при изображении без текста возвращает пустой/пробельный результат без исключения.
- **Связь с ADR:** ADR-002, Решение 1 и 4.

### Задача 2: Хендлер изображений (`handlers/image.py`)
- **Что:** роутер aiogram, принимающий:
  - фото (`message.photo`) — ФТ-1;
  - изображение, отправленное как файл-документ (`message.document` с image-MIME) — ФТ-2;
  скачивающий файл штатным клиентом aiogram (`bot.download` / `bot.get_file` + download) и
  вызывающий `services/ocr.py`. При непустом результате отправляет пользователю **сырой**
  распознанный текст; при пустом — понятное русскоязычное сообщение, что текст не распознан (ФТ-6).
- **Где (файлы/модули):** `handlers/image.py`; экспорт роутера через `handlers/__init__.py`.
- **Контракт:**
  - фото берётся в максимальном доступном размере (последний элемент `message.photo`);
  - документ обрабатывается только при image-MIME (например `mime_type`, начинающийся с `image/`);
    не-image документы этим роутером не перехватываются;
  - скачивание — только штатным механизмом aiogram, без кастомных URL/хостов (ADR-002, Решение 6,
    риск Р-1);
  - ответ при непустом OCR — ровно сырой текст без форматирования/структурирования (ФТ-5, КП-4);
  - тексты системных сообщений — на русском (НФТ-3).
- **Критерии готовности:** оба типа вложений (фото и image-документ) доходят до OCR и дают ответ;
  пустой результат → сообщение о нераспознанном тексте; не-image документы хендлером не
  перехватываются.
- **Связь с ADR:** ADR-002, Решение 4 и 6.

### Задача 3: Регистрация роутера и порядок обработки
- **Что:** зарегистрировать роутер изображений в `bot.py`. Обеспечить, чтобы сообщение с фото/
  image-документом обрабатывалось OCR-хендлером, а не текстовой заглушкой `handlers/text.py`
  (её фильтр `~Command("start","help")` перехватывает любое не-командное сообщение, включая фото
  без подписи).
- **Где (файлы/модули):** `bot.py`, `handlers/__init__.py`; при необходимости — `handlers/text.py`
  (сужение фильтра, чтобы не ловить фото/image-документы).
- **Контракт:** роутер изображений включается в диспетчер **до** текстового роутера, либо текстовый
  фильтр сужается. Текстовая заглушка на обычный текст (без вложения) сохраняется без изменений
  поведения.
- **Критерии готовности:** изображение обрабатывается OCR-хендлером (не заглушкой); обычный текст
  по-прежнему получает заглушку; команды `/start`, `/help` работают как раньше.
- **Связь с ADR:** ADR-002, Решение 5.

### Задача 4: Зависимости (`requirements.txt`)
- **Что:** добавить `pytesseract` и `Pillow`.
- **Где (файлы/модули):** `requirements.txt`.
- **Контракт:** добавляются только `pytesseract` и `Pillow`. НЕ добавлять faster-whisper, ffmpeg-
  обёртки, Groq SDK (КП-9).
- **Критерии готовности:** `pip install -r requirements.txt` ставит OCR-зависимости и не тянет
  аудио/Groq.
- **Связь с ADR:** ADR-002, «Влияние на код».

### Задача 5: Установка Tesseract в Docker-образ (`Dockerfile`)
- **Что:** в **runtime**-стейдж (финальный `FROM python:3.12-slim`) добавить установку системных
  пакетов `tesseract-ocr tesseract-ocr-rus tesseract-ocr-eng` через `apt-get` с чисткой кэша apt
  (`rm -rf /var/lib/apt/lists/*` в том же слое). Установку выполнить **до** переключения на
  `USER appuser`.
- **Где (файлы/модули):** `Dockerfile`.
- **Контракт:** пакеты Tesseract ставятся именно в runtime-стейдж (в builder-стейдже они бесполезны —
  туда не копируется бинарь, см. ADR-002 Решение 2). Сохраняются multi-stage, non-root (`USER
  appuser`), запуск `python bot.py`, отсутствие `EXPOSE`/портов. Профиль поставки ADR-001 не
  меняется. Новых env-переменных нет → `deploy/docker-compose.yml`, `deploy/.env.example`,
  `.env.example`, `.github/workflows/deploy.yml` править **не нужно** (зафиксировать в чек-листе как
  «без изменений»).
- **Критерии готовности:** в собранном образе доступен бинарь `tesseract` и языковые данные
  `rus`+`eng`; OCR работает внутри контейнера (КП-7); образ остаётся non-root; кэш apt очищен.
- **Связь с ADR:** ADR-002, Решение 2; деплой-контракт — ADR-001 (профиль «бот без HTTP-порта»),
  `deploy-guide.md`.

## Тесты

Минимальный обязательный набор (`pytest`; сетевые вызовы к Telegram не выполняются —
заглушки/фикстуры):

- **Т-1. OCR на изображении с текстом:** сгенерировать в тесте изображение с русским и/или
  английским текстом (например через `Pillow`) и убедиться, что `services/ocr.py` возвращает
  непустой текст, содержащий ожидаемые слова. (КП-1, КП-3, ФТ-4)
- **Т-2. OCR — пустой результат:** на изображении без текста (например однотонный фон) сервис
  возвращает пустой/пробельный результат без исключения. (КП-5, ФТ-6)
- **Т-3. Регистрация image-роутера:** роутер изображений фактически зарегистрирован в диспетчере и
  хендлеры фото/image-документа подключены; порядок относительно текстового роутера обеспечивает,
  что изображение не перехватывается заглушкой. (КП-6, ADR-002 Решение 5)

> Тесты Т-1/Т-2 требуют установленного Tesseract с `rus`+`eng` в окружении прогона. Если в среде
> тестов Tesseract недоступен — тест помечается `skip` с явной причиной (а не удаляется); в CI
> прогон OCR-тестов должен идти на образе/окружении с установленным Tesseract.

## Риски и митигации

- **Р-1. Скачивание файла завязано на пиннинг `api.telegram.org`** (`extra_hosts` в compose). Если
  пиннинг отвалится (сменится рабочий Telegram-DC), сломается и long-polling, и download файлов. →
  **Митигация:** скачивать только штатным клиентом aiogram через `api.telegram.org` (без кастомных
  хостов) — тогда download использует тот же существующий пиннинг; при сбое подобрать рабочий DC
  в `extra_hosts` (как уже отмечено в комментарии compose). Отдельных действий в этом релизе не
  требует.
- **Р-2. Конкуренция за CPU.** На ivahub 1 vCPU; несколько одновременных тяжёлых изображений делят
  процессор, время ответа растёт. → **Митигация:** `asyncio.to_thread` сохраняет отзывчивость event
  loop; ограничение параллелизма/очередь отнесены к v0.5.0. Приемлемо для текущих объёмов.
- **Р-3. Рост размера образа** из-за Tesseract и языковых данных. → **Митигация:** чистка кэша apt
  в том же слое; на ivahub свободно ~22 GB — некритично.
- **Р-4. Tesseract в builder-стейдже вместо runtime** → `pytesseract` не найдёт бинарь в рантайме. →
  **Митигация:** Задача 5 явно требует установку в runtime-стейдж; арх-контроль и ревью проверяют.
- **Р-5. Просачивание тяжёлых зависимостей** (faster-whisper/ffmpeg/Groq). → **Митигация:** КП-9 и
  контракт Задачи 4 запрещают; арх-контроль и ревью проверяют состав `requirements.txt`/образа.
- **Р-6. Изображение перехватывает текстовая заглушка** (фильтр `~Command` в `handlers/text.py`). →
  **Митигация:** Задача 3 фиксирует порядок роутеров / сужение фильтра; Т-3 проверяет.
