# Спецификация релиза v0.8.0

- **Дата планирования:** 2026-06-23
- **Тип релиза:** minor (обратимо-совместимое добавление функциональности; публичный контракт канала аудио сохраняется)
- **Цели:** перевести основную транскрипцию аудио на Cloudflare Workers AI `whisper-large-v3-turbo` (точнее, быстрее, разгружает CPU/RAM ivahub); локальный faster-whisper остаётся фолбэком и режимом `ASR_PROVIDER=local`; сбой облака не ломает ответ.

## Источники

- BA-требования: `ba-req/ba-requirements-008.md`
- ADR: `adrs/adr-008.md` (основной), `adrs/adr-003.md` (локальная транскрипция — дополняется), `adrs/adr-005.md` (образец CF-провайдера `httpx`), `adrs/adr-001.md` (профиль поставки), `adrs/adr-007.md` (отдача результата)

## Объём релиза

### Задача 1: Рефактор `services/transcribe.py` в оркестратор + провайдеры с фолбэком

- **Что:** превратить сервис в оркестратор «аудио → текст» с абстракцией провайдера (по образцу
  ADR-005, Решение 1). Облачный провайдер Cloudflare Whisper (дефолт) через `httpx`; локальный
  faster-whisper (существующий путь) через `asyncio.to_thread`. Выбор провайдера по `ASR_PROVIDER`.
  Фолбэк на локальную модель при любом сбое облака. Порог размера для крупных файлов. Логирование
  факта фолбэка.
- **Где (файлы/модули):** `services/transcribe.py`.
- **Контракт:**
  - Публичная корутина `async def transcribe(audio_bytes: bytes) -> str` — **сигнатура сохраняется**
    (вызывается из `handlers/audio.py`).
  - Облачный провайдер: `POST https://api.cloudflare.com/client/v4/accounts/{CF_ACCOUNT_ID}/ai/run/{CF_WHISPER_MODEL}`,
    заголовок `Authorization: Bearer {CF_API_TOKEN}`, тело `{"audio": "<base64(audio_bytes)>"}`,
    результат из `result.text` (`.strip()`).
  - Константы модуля: `_CF_TIMEOUT = 60.0`, `_CF_MAX_AUDIO_BYTES = 8 * 1024 * 1024`.
  - Локальный путь: существующие `_get_model()` / `_transcribe_sync()` сохраняются; локальная
    обёртка вызывает их через `asyncio.to_thread` (как сейчас).
  - Облачный путь — нативный async `httpx.AsyncClient` с `await` (без `to_thread`).
  - `CF_ACCOUNT_ID` / `CF_API_TOKEN` читаются из окружения (как в `services/structure.py`).
  - `ASR_PROVIDER` и `CF_WHISPER_MODEL` читаются из окружения/конфига; дефолты — `cloudflare` и
    `@cf/openai/whisper-large-v3-turbo`.
- **Поведение оркестратора:**
  1. Если `ASR_PROVIDER == "local"` → только локальный путь, к Cloudflare не обращаться.
  2. Иначе (cloudflare / незадано / неизвестное значение):
     - если `len(audio_bytes) > _CF_MAX_AUDIO_BYTES` → лог (info/warning «exceeds CF size threshold,
       using local») и локальный путь;
     - если CF-кредов нет → лог (warning) и локальный путь (фолбэк);
     - иначе попытка облачного пути (единичная, `timeout=60`); при успехе и непустом тексте — вернуть
       `result.text.strip()`; при **любом** сбое (исключение `httpx`, не-2xx через `raise_for_status`,
       некорректный/пустой ответ) — `logger.warning`/`logger.exception` с причиной + фолбэк на
       локальный путь.
  3. Пустой результат распознавания (и облако, и локально) возвращается как пустая строка — решение
     «речь не распознана» остаётся в хендлере (не меняется).
- **Критерии готовности:**
  - `transcribe` сохраняет сигнатуру и возвращает `str`.
  - При дефолте/`cloudflare` запрос идёт на CF-эндпоинт с `whisper-large-v3-turbo`, тело —
    `{"audio": <base64>}`, текст берётся из `result.text`.
  - При `ASR_PROVIDER=local` Cloudflare не вызывается.
  - Любой облачный сбой → фолбэк на локальную модель, исключение не пробрасывается, факт фолбэка
    логируется.
  - Крупный файл (>8 МБ) при дефолте идёт локально без обращения к Cloudflare.
  - Облачный путь использует `httpx.AsyncClient` и `await`, не `asyncio.to_thread`.
- **Связь с ADR:** ADR-008, Решения 1–8.

### Задача 2: Конфиг — `ASR_PROVIDER` и `CF_WHISPER_MODEL`

- **Что:** добавить чтение новых переменных с дефолтами; без обязательной валидации (в отличие от
  `BOT_TOKEN`). `CF_WHISPER_MODEL` — **отдельно** от `CF_MODEL` структурирования.
- **Где (файлы/модули):** `config.py`.
- **Контракт:**
  - `ASR_PROVIDER` — дефолт `cloudflare`; нормализация (`.strip().lower()`); неизвестное значение
    трактуется как `cloudflare` (на уровне оркестратора).
  - `CF_WHISPER_MODEL` — дефолт `@cf/openai/whisper-large-v3-turbo`.
  - Допустимо читать значения непосредственно в `services/transcribe.py` из окружения (как
    `structure.py` читает свои), либо положить в `config`-словарь — на усмотрение разработчика, но
    единообразно с тем, как сделано для структурирования. Главное: дефолты работают, переключение —
    конфигом без правки кода.
- **Критерии готовности:** при незаданных переменных используются дефолты; бот стартует без новых
  переменных в `.env`; задание `ASR_PROVIDER`/`CF_WHISPER_MODEL` меняет поведение без правки кода;
  `CF_WHISPER_MODEL` не пересекается с `CF_MODEL`.
- **Связь с ADR:** ADR-008, Решения 6–7, 10.

## Изменения публичного контракта

Внешний контракт канала аудио **не меняется**: на вход те же `message.voice` / `message.audio` /
audio-документ, на выходе тот же транскрипт через `send_result` (ADR-007). Меняется **источник**
транскрипта (Cloudflare по умолчанию; локально — фолбэк/режим). Внутренняя сигнатура
`transcribe(audio_bytes) -> str` сохраняется. Новые env-переменные (`ASR_PROVIDER`,
`CF_WHISPER_MODEL`) имеют рабочие дефолты — миграция не требуется. Новых обязательных секретов нет
(CF-креды уже есть). Изменение обратимо-совместимое (minor).

## Тесты

Все тесты — **на моках, без реальной сети и без реальной модели** (мок `httpx.AsyncClient.post` для
облака; мок `_get_model`/`WhisperModel` для локального). Файл: `tests/test_transcribe.py`
(дополнить; существующие тесты локального пути сохранить). Минимальный набор:

- **CF-провайдер, успех:** мок `httpx` возвращает `{"result": {"text": "распознанный текст", "transcription_info": {"language": "ru"}}}` → `transcribe` возвращает `"распознанный текст"`; запрос ушёл на CF-эндпоинт с `accounts/.../ai/run/@cf/openai/whisper-large-v3-turbo`, заголовок `Authorization: Bearer <token>`, тело содержит ключ `audio` с base64 переданных байтов.
- **Парсинг `result.text`:** результат берётся именно из `result.text` (а не из `result.response`).
- **Фолбэк на исключении `httpx`:** `post` бросает `httpx.ConnectError`/`httpx.TimeoutException` → вызывается локальный путь (мок `_get_model`), возвращается локальный результат; исключение не пробрасывается.
- **Фолбэк на не-2xx:** `raise_for_status` бросает `HTTPStatusError` (status 500/429) → фолбэк на локальный путь.
- **Фолбэк при отсутствии CF-кредов:** `CF_ACCOUNT_ID`/`CF_API_TOKEN` пусты, `ASR_PROVIDER=cloudflare` → Cloudflare не вызывается (`post` не вызван), идёт локальный путь.
- **Выбор провайдера `ASR_PROVIDER=local`:** Cloudflare не вызывается (`post` не вызван), используется только локальный путь.
- **Дефолтный провайдер — cloudflare:** при незаданном `ASR_PROVIDER` (с заданными CF-кредами) запрос идёт на CF-эндпоинт.
- **Пустой текст:** `result.text` пуст/пробельный → `transcribe` возвращает пустую строку (хендлер сам решит про «речь не распознана»); либо, при трактовке пустого ответа как сбоя, происходит фолбэк на локальный путь — поведение зафиксировать тестом по реализации (главное: без падения и корректный тип `str`).
- **Крупный файл → локальный путь:** `len(audio_bytes) > _CF_MAX_AUDIO_BYTES` при дефолте → `post` не вызван, используется локальный путь.
- **Неблокирование облачного пути:** облачный путь использует `httpx.AsyncClient`/`await`, в исходнике облачной ветки нет `to_thread` (по образцу `test_structure_text_uses_httpx_async_client`).
- **Сохранение локального контракта:** существующие тесты локального пути (`test_transcribe_returns_joined_segments` и др.) продолжают проходить в режиме `local`/фолбэка.

## Риски и митигации

- **Риск:** лимиты бесплатного tier Cloudflare исчерпаны (429) → **Митигация:** трактуется как сбой → фолбэк на локальную модель; единичная попытка без ретраев бережёт лимит (ADR-008, Решение 4).
- **Риск:** крупный файл не укладывается в лимит размера запроса (base64 +33 %) → **Митигация:** порог `_CF_MAX_AUDIO_BYTES = 8 МБ`, крупные файлы сразу локально (ADR-008, Решение 5).
- **Риск:** задержка/повисание сети → **Митигация:** явный `timeout=60` + фолбэк; event loop не блокируется (нативный async).
- **Риск:** фолбэк **маскирует деградацию качества** (turbo → локальная `base`) — пользователь и оператор не видят, что облако отказало → **Митигация:** обязательное логирование факта фолбэка с причиной (`logger.warning`/`exception`); деградация видна в логах (ADR-008, Решение 8).
- **Риск:** достижимость CF Whisper с прода не подтверждена на реальном аудио → **Митигация:** проверка при деплое (КП-12, ОВ-3); достижимость уже подтверждена вживую (HTTP 200), риск низкий.
