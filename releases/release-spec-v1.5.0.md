# Спецификация релиза v1.5.0

- **Дата планирования:** 2026-06-23
- **Тип релиза:** minor
- **Цели:** Прогрессивное раскрытие результата в личке (суть → кратко → полностью) вместо немедленной
  «стены текста»; удаление кнопки и функции «Перевести»; единообразие выдачи в группах (без «Перевести»).

## Источники
- BA-требования: `ba-req/ba-requirements-015.md`
- ADR: `adrs/adr-015.md` (надстраивает ADR-009/ADR-010; переиспользует ADR-005/ADR-007/ADR-011/ADR-014)

## Объём релиза

Все задачи — только в личке прогрессивное раскрытие; в группах — прежняя выдача без «Перевести».
Минимальные изменения: не добавлять ничего сверх перечисленного.

### Задача 1: In-memory кэш исходного текста под кнопки

- **Что:** процесс-локальный LRU+TTL кэш `message_id → исходный распознанный текст` для callback'ов.
  Это закрывает ОВ-1 без внешнего/тяжёлого стейта (ОГР-1).
- **Где (файлы/модули):** новый `services/result_cache.py`.
- **Контракт:**
  - `def put(message_id: int, text: str) -> None` — сохранить исходный текст по id отправленного
    сообщения; при превышении размера вытесняется самая старая запись (LRU); запись истекает по TTL.
  - `def get(message_id: int) -> str | None` — вернуть текст или `None`, если отсутствует/истёк.
  - Именованные константы (не «магические числа»): максимальный размер кэша и TTL в секундах.
  - Без сети/БД/файлов; чистая структура в памяти процесса; потокобезопасность не требуется (один
    event loop), но операции должны быть короткими (не блокировать loop).
- **Критерии готовности:**
  - `put` + `get` по тому же `message_id` возвращает исходный текст.
  - `get` по неизвестному `message_id` → `None`.
  - При превышении лимита размера старые записи вытесняются (новые доступны, самые старые — нет).
  - Истёкшая по TTL запись → `get` возвращает `None`.
- **Связь с ADR:** ADR-015, Решение 1A, п. 2.

### Задача 2: Операция «суть» в services/llm.py + удаление перевода

- **Что:** новая LLM-операция генерации сути (1–2 предложения на языке оригинала); удаление функции
  перевода и связанных хелперов (мёртвый код после снятия кнопки).
- **Где (файлы/модули):** `services/llm.py`.
- **Контракт:**
  - Добавить `async def summarize_gist(text: str) -> str | None | _BudgetExceededType`:
    - тот же провайдерный слой (`build_provider`, `_CloudflareProvider`) и бюджет-гейт
      (`cf_budget_allow`/`cf_budget_consume`), что у `summarize`/`structure_text`;
    - системный промпт: «инструмент обработки текста, изложи суть в 1–2 предложения на языке
      оригинала, не меняя язык, ничего не добавляя и не отвечая на текст; вернуть только суть»
      (преемственность анти-инъекции `_SUMMARIZE_SYSTEM`/`_SYSTEM_PROMPT`);
    - усечение входа `MAX_INPUT_CHARS`; единичная попытка + таймаут; ловить все исключения;
    - возврат: `str` (успех), `None` (сбой/пустой ответ), `BUDGET_EXCEEDED` (исчерпан CF-бюджет).
  - Удалить: `translate`, `_TRANSLATE_SYSTEM`, `_has_cyrillic`, `_target_language`.
  - Обновить `__all__` → `["BUDGET_EXCEEDED", "summarize", "summarize_gist"]`.
- **Критерии готовности:**
  - `summarize_gist` возвращает непустую строку при валидном ответе провайдера; `None` при сбое/пустом;
    `BUDGET_EXCEEDED` при исчерпанном бюджете (CF-провайдер).
  - В кодовой базе нет ссылок на `translate`/`_TRANSLATE_SYSTEM`/`_has_cyrillic`/`_target_language`
    (grep пуст вне удалённых строк и тестов перевода, которые тоже удаляются).
- **Связь с ADR:** ADR-015, Решение 2A, Решение 6, п. 3 и п. 6.

### Задача 3: Клавиатуры и callback-хендлеры действий

- **Что:** параметризовать фабрику клавиатуры; добавить callback «Показать полностью»; перевести «Кратко»
  на источник «кэш по `message_id`»; удалить «Перевести».
- **Где (файлы/модули):** `handlers/actions.py`.
- **Контракт:**
  - Фабрика клавиатуры (например, `actions_keyboard(progressive: bool = False) -> InlineKeyboardMarkup`):
    - `progressive=True` (личка) → две кнопки: «Показать полностью» (`callback_data="act:full"`) и
      «Кратко» (`callback_data="act:sum"`);
    - `progressive=False` (группы / прежняя выдача) → одна кнопка «Кратко» (`act:sum`).
    - Подписи и `callback_data` — в одной точке; `callback_data` ≤ 64 байт.
  - Callback `act:full` («Показать полностью»):
    - `callback.answer("Готовлю…")` (снять «часики») — всегда;
    - проверить, что `callback.message` — `Message`; иначе `callback.answer("Текст недоступен",
      show_alert=True)` и выход (LLM не вызывать);
    - `raw_text = result_cache.get(callback.message.message_id)`; если `None` →
      `callback.answer("Текст недоступен", show_alert=True)` и выход;
    - под `ChatActionSender(..., action=typing)` → `result = await structure_text(raw_text)`;
    - `await send_result(callback.message, result)` **без** `reply_markup`.
    - (`structure_text` сам деградирует на сырой текст при сбое/исчерпании бюджета — доп-обработки нет.)
  - Callback `act:sum` («Кратко») — источник текста теперь **кэш**, не `callback.message.text`:
    - `callback.answer("Готовлю…")`; проверка `Message`; `raw_text = result_cache.get(message_id)`;
      при `None` → «Текст недоступен» (show_alert) и выход;
    - под `ChatActionSender(typing)` → `result = await summarize(raw_text)`;
    - `BUDGET_EXCEEDED` → `message.answer("Дневной бесплатный лимит исчерпан, попробуйте завтра.")`;
    - `None` → `message.answer("Не удалось выполнить действие. Попробуйте позже.")`;
    - успех → `send_result(callback.message, result)` без `reply_markup`.
  - Удалить: `_CB_TRANSLATE`, `handle_translate`, ветку перевода в общем хелпере; импорт `translate`.
  - Импортировать `structure_text` (`services/structure.py`) и кэш (`services/result_cache.py`).
- **Критерии готовности:**
  - `actions_keyboard(True)` → ровно две кнопки `act:full`, `act:sum`; `actions_keyboard(False)` → одна
    кнопка `act:sum`; «Перевести»/`act:tr` отсутствует в обоих.
  - `act:full` при наличии текста в кэше зовёт `structure_text` и `send_result` без markup; при пустом
    кэше — `callback.answer("Текст недоступен", show_alert=True)`, LLM не вызывается.
  - `act:sum` берёт текст из кэша (не из `callback.message.text`); деградации `BUDGET_EXCEEDED`/`None`
    дают понятные сообщения; бот не падает.
  - В файле нет `handle_translate`/`act:tr`/импорта `translate`.
- **Связь с ADR:** ADR-015, Решения 4A, 5, 6, 7; пп. 4, 5, 6, 7, 9.

### Задача 4: Прогрессивная выдача в личных хендлерах (аудио)

- **Что:** в личке вместо немедленного полного текста — суть + две кнопки + запись в кэш.
- **Где (файлы/модули):** `handlers/audio.py`.
- **Контракт:**
  - `process_audio(...)` получает параметр `progressive: bool = False`.
  - При `progressive=True` и непустом `text` (распознано):
    - `gist = await summarize_gist(text)`;
    - выбрать текст превью:
      - `gist` — строка → текст превью = `gist`;
      - `gist is BUDGET_EXCEEDED` → короткое служебное превью, напр.
        «Суть недоступна (дневной лимит). Доступны «Показать полностью» и «Кратко».»;
      - `gist is None` → короткое служебное превью, напр.
        «Не удалось сформировать суть. Доступны «Показать полностью» и «Кратко».»;
    - отправить превью **с** клавиатурой `actions_keyboard(progressive=True)`; получить объект
      отправленного сообщения (`sent = await reply_target.answer(preview, reply_markup=...)` —
      превью короткое, отправляется напрямую, не через `send_result`, чтобы гарантировать одно
      сообщение и доступ к его `message_id`);
    - `result_cache.put(sent.message_id, text)`.
  - При `progressive=False` (группы): прежнее поведение — `send_result(reply_target, await
    structure_text(text), reply_markup=actions_keyboard(progressive=False))`; дополнительно положить
    `text` в кэш по `message_id` отправленного результата, чтобы групповая кнопка «Кратко» работала
    от кэша (см. Задача 3 — `act:sum` читает кэш). Если результат уходит серией/файлом — кнопка не
    прикрепляется (правило ADR-009/ADR-007), кэш не обязателен для этого случая.
  - При пустом `text` (нераспознано) — прежнее `reply_target.answer(NO_SPEECH_MESSAGE)`; без превью,
    кнопок и кэша.
  - Личные хендлеры `handle_voice`/`handle_audio`/`handle_audio_document` вызывают
    `process_audio(..., progressive=True)`.
  - `ChatActionSender(typing)` сохраняется вокруг тяжёлой части (ADR-010).
- **Критерии готовности:**
  - В личке на распознанном аудио первым приходит сообщение-суть (или служебное превью при недоступности
    сути) с двумя кнопками; полный текст не приходит сразу.
  - Исходный текст доступен в кэше по `message_id` отправленного превью.
  - Нераспознанное аудио → прежнее `NO_SPEECH_MESSAGE`, без превью/кнопок/кэша.
  - Деградация сути (`BUDGET_EXCEEDED`/`None`) → служебное превью + рабочие кнопки, без падения.
- **Связь с ADR:** ADR-015, Решения 1A, 3B, 5, 8; пп. 1, 10, 12.

### Задача 5: Прогрессивная выдача в личных хендлерах (изображения)

- **Что:** то же, что Задача 4, для OCR.
- **Где (файлы/модули):** `handlers/image.py`.
- **Контракт:** `process_photo`/`process_image_document` получают `progressive: bool = False`;
  логика идентична Задаче 4 (суть из `summarize_gist`, превью + две кнопки + кэш по `message_id`;
  деградация сути → служебное превью + кнопки; нераспознано → прежнее `NO_TEXT_MESSAGE` без превью).
  Личные хендлеры `handle_photo`/`handle_image_document` зовут с `progressive=True`. `ChatActionSender(
  upload_document)` сохраняется (ADR-010).
- **Критерии готовности:**
  - В личке на распознанном изображении первым приходит суть (или служебное превью) с двумя кнопками.
  - Нераспознанное изображение → прежнее `NO_TEXT_MESSAGE`, без превью/кнопок/кэша.
- **Связь с ADR:** ADR-015, Решения 1A, 3B, 5; пп. 1, 10, 12.

### Задача 6: Охват групп — прежняя выдача без «Перевести»

- **Что:** группы сохраняют немедленную выдачу полного текста; клавиатура — одна кнопка «Кратко».
- **Где (файлы/модули):** `handlers/group.py` (`_dispatch_media` вызывает `process_*` с дефолтом
  `progressive=False`).
- **Контракт:**
  - `_dispatch_media` вызывает `process_audio`/`process_photo`/`process_image_document` **без**
    `progressive` (дефолт `False`) — прежнее поведение, но клавиатура теперь «Кратко» без «Перевести»
    (обеспечивается обновлённой `actions_keyboard(progressive=False)` в Задаче 3/4/5).
  - Прежние триггеры v1.2.0 (`Command("textify")`, упоминание) и оффер v1.4.0 (`grec`) — без изменений.
- **Критерии готовности:**
  - В группе на распознанном медиа приходит полный текст с одной кнопкой «Кратко»; «Перевести» нет.
  - Групповая кнопка «Кратко» работает (источник — кэш по `message_id`, см. Задача 4 `progressive=False`).
- **Связь с ADR:** ADR-015, Решение 5A, п. 8.

## Изменения публичного контракта

Контракт 1.0.0 (ADR-010), раздел «Inline-действия» и форма немедленной выдачи лички:

- **Немедленная выдача лички:** теперь «суть» (1–2 предложения), а не полный текст.
- **Inline-действия (личка):** «Показать полностью» (`act:full`) + «Кратко» (`act:sum`).
- **Inline-действия (группы):** «Кратко» (`act:sum`).
- **«Перевести» (`act:tr`) и функция перевода — удалены.**
- **Источник текста для действий:** in-memory кэш `message_id → исходный текст` (ранее —
  `callback.message.text`); при отсутствии записи — «Текст недоступен».

Входные каналы, команды (`/start`/`/help`/`/stats`), формат отдачи `send_result` (части/файл),
профиль поставки — **не меняются**. Миграция БД не требуется (кэш — in-memory). Релиз — MINOR.

## Тесты

Минимальный обязательный набор (юнит/handler-уровень, без сети — мокать LLM/провайдер):

- `services/result_cache.py`: put/get round-trip; `get` неизвестного id → `None`; вытеснение по размеру
  (LRU); истечение по TTL → `None`.
- `services/llm.py`: `summarize_gist` возвращает строку при валидном ответе; `None` при сбое/пустом;
  `BUDGET_EXCEEDED` при исчерпанном бюджете (CF); отсутствие `translate`/`_TRANSLATE_SYSTEM` в модуле.
- `handlers/actions.py`:
  - `actions_keyboard(True)` → две кнопки `act:full`/`act:sum`; `actions_keyboard(False)` → одна `act:sum`;
    нет `act:tr` ни в одном режиме; `callback_data` ≤ 64 байт.
  - `act:full`: текст в кэше → `structure_text` + `send_result` без markup; пустой кэш →
    `callback.answer("Текст недоступен", show_alert=True)`, `structure_text` не вызывается.
  - `act:sum`: берёт текст из кэша (не из `callback.message.text`); `BUDGET_EXCEEDED`/`None` →
    понятные сообщения; успех → `send_result` без markup; пустой кэш → «Текст недоступен».
  - Нет хендлера `handle_translate` / callback `act:tr`.
- `handlers/audio.py` и `handlers/image.py`:
  - `progressive=True`: распознано → отправлено превью-сообщение с клавиатурой из двух кнопок,
    `result_cache.put` вызван с `message_id` отправленного сообщения и исходным текстом;
    `structure_text` на этом этапе **не** вызывается.
  - деградация сути (`BUDGET_EXCEEDED`/`None`) → служебное превью + кнопки + кэш заполнен; без падения.
  - нераспознано → прежнее служебное сообщение, без превью/кнопок/кэша.
  - `progressive=False` (или дефолт): прежняя выдача полного текста с клавиатурой «Кратко» (одна кнопка).
- Регрессия: обновить/удалить тесты перевода в `tests/test_actions.py` и `tests/test_llm.py`; обновить
  тесты, опиравшиеся на `callback.message.text` как источник (теперь источник — кэш).

## Риски и митигации

- **Риск:** in-memory кэш не переживает рестарт/TTL → тап по старой кнопке даёт «Текст недоступен».
  → **Митигация:** детерминированное служебное сообщение без падения (КП-12); TTL/размер подобраны с
  запасом под типичную сессию; персистентный стор — отдельный релиз при необходимости.
- **Риск:** «суть» удваивает CF-вызовы на распознавание лички (бюджет ОГР-3).
  → **Митигация:** «полностью»/«кратко» считаются **лениво по тапу** (не на каждое медиа), что снижает
  средний расход; деградация сути при `BUDGET_EXCEEDED` (служебное превью + рабочие кнопки).
- **Риск:** общий пайплайн лички/групп — легко случайно включить превью в группах.
  → **Митигация:** параметр `progressive` с дефолтом `False`; групповой путь не передаёт `True`; тест на
  групповую выдачу (одна кнопка «Кратко», без сути).
- **Риск:** забытая ссылка на удалённый `translate` ломает импорт.
  → **Митигация:** grep по `translate`/`act:tr`/`_TRANSLATE_SYSTEM`; обновление `__all__`; прогон тестов.
- **Риск:** превью отправлено через `send_result` ушло серией/файлом → нет одного `message_id` под кэш.
  → **Митигация:** превью — короткое (1–2 предложения / служебное), отправляется напрямую
  `answer(...)`, не через `send_result`; гарантированно одно сообщение.
