# requirements/

Каталог сырых требований от пользователя.

## Формат имени
`req-<short-name>-NNN.md`, где `NNN` — сквозной трёхзначный номер.

## Содержимое
Свободная форма. Цель — зафиксировать то, что пользователь хочет, до того как бизнес-аналитик начнёт это структурировать. Чем подробнее — тем меньше открытых вопросов потом.

## Дальше
`/ba NNN` → бизнес-аналитик создаст `ba-req/ba-requirements-NNN.md`.
