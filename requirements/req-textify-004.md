# req-textify-004: Улучшение качества OCR (v0.4.0)

## Что нужно
Повысить качество распознавания текста на изображениях за счёт предобработки картинки
перед Tesseract и более удачных настроек движка. Контракт не меняется — на вход то же фото,
на выходе тот же текст, просто точнее.

Конкретно (предобработка через OpenCV):
- Исправление ориентации по EXIF (фото с телефона часто повёрнуты).
- Перевод в grayscale.
- Апскейл мелких изображений до разумного размера (Tesseract точнее на ~300 DPI).
- Адаптивная бинаризация (отделить текст от фона при неравномерном освещении).
- Деноизинг (с оглядкой на 1 vCPU — выбрать нетяжёлую операцию).
- Устранение наклона (deskew).
- Движок Tesseract `--oem 1` (LSTM), подбор `--psm` (обосновать выбор).
- Языки rus+eng — без изменений.

UX-подсказка (бесплатный выигрыш): подсказать пользователю присылать изображение **как файл
(документ)**, а не «фото» — тогда Telegram не пересжимает его в JPEG и качество выше.
Хендлер документов-изображений уже есть; добавить подсказку (например, в /help и/или в
сообщение при нераспознанном тексте) — без навязчивости.

## Ограничения окружения (важно)
- Сервер ivahub: **1 vCPU, 1.9 GB RAM**. Предобработка тоже CPU-bound — выполнять вне event
  loop (`asyncio.to_thread`, как уже сделано для OCR). Тяжёлые операции (например,
  fastNlMeansDenoising на больших картинках) могут заметно тормозить — выбрать
  баланс «качество/скорость», при необходимости ограничивать размер обрабатываемого
  изображения сверху.
- Новая зависимость: `opencv-python-headless` (+ `numpy`, обычно уже транзитивно). Для
  работы в slim-образе могут понадобиться системные библиотеки (например `libglib2.0-0`,
  `libgomp1`) — добавить в Dockerfile, проверить запуск `import cv2` в образе.
- Образ подрастёт — для ivahub (22 GB свободно) приемлемо.

## Зачем
OCR (v0.2.0) сейчас работает без подготовки картинки — это базовое качество. Пользователь
запросил повышение точности распознавания изображений. Предобработка даёт основной прирост.
