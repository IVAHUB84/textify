# req-textify-005: Структурирование текста через бесплатный LLM (v0.5.0)

## Что нужно
Привести распознанный текст (из OCR и из транскрипции аудио) к читаемому
структурированному виду — заголовки, списки, ключевые пункты в Markdown — с помощью
бесплатного LLM API. Это исходная цель v0.5.0, но **без Groq**: Groq гео-блокирует
российские IP, с сервера ivahub недоступен.

Контракт результата для пользователя: на вход голос/аудио или фото, на выходе — не сырой
текст, а аккуратно оформленный (структура поверх распознавания). Подключается к **обоим**
входным каналам (картинки и аудио) — общий сквозной сценарий.

Конкретно:
- `services/structure.py` — обёртка над LLM-провайдером: принимает сырой текст,
  возвращает структурированный Markdown (заголовки/списки/ключевые пункты).
- **Провайдер по умолчанию — Cloudflare Workers AI** (бесплатный tier, RU-IP не блокирует,
  работает с ivahub напрямую без прокси). Модель — лёгкая мультиязычная (Llama 3.x / Qwen),
  выбор и параметры обосновать в ADR.
- **Абстракция провайдера через env** (`LLM_PROVIDER`): Cloudflare — дефолт, Groq —
  опция на будущее (когда решится доступность). Без переписывания кода при смене.
- Промпт ориентирован на rus+eng контент, сохраняет смысл, не выдумывает факты,
  не дописывает то, чего нет в исходнике.
- Вызов — неблокирующий (сетевой I/O, не CPU): корректная интеграция в event loop aiogram.
- Секреты провайдера — только через `.env`: `CF_ACCOUNT_ID`, `CF_API_TOKEN`
  (для Groq-ветки — `GROQ_API_KEY`), плейсхолдеры в `.env.example`. Никогда не хардкодить.

## Фолбэк (важно)
- При недоступности/ошибке/таймауте LLM — **отдавать сырой распознанный текст**, а не падать.
  Структурирование — улучшение поверх, не критичный путь.

## Ограничения окружения
- Сервер ivahub: 1 vCPU, 1.9 GB RAM. Задача сетевая (не CPU-bound) — нагрузки на CPU почти
  не добавляет. Главное — не блокировать event loop на время HTTP-запроса.
- Требуется только бесплатный аккаунт Cloudflare (`ACCOUNT_ID` + `API_TOKEN`), без привязки
  карты. Аккаунт и токен заводит пользователь.
- Учесть суточные лимиты бесплатного tier и размер запроса (очень длинные транскрипты —
  при необходимости усечь/разбить, обосновать в ADR).

## Зачем
OCR (v0.4.0) и транскрипция (v0.3.0) сейчас отдают сырой текст. Структурирование — заявленная
ценность продукта (см. README/roadmap). Groq для этого недоступен из-за гео-блока, поэтому
берём бесплатного провайдера, работающего с RU-IP. Пользователь выбрал полностью бесплатный
вариант без прокси.
