# req-textify-006: Учёт пользователей и команда /stats (v0.6.0)

## Что нужно
Дать возможность узнавать, сколько у бота уникальных пользователей и как им пользуются.
Telegram не отдаёт «число подписчиков» бота — поэтому считаем сами.

Конкретно:
- При каждом обращении фиксировать пользователя: `user_id` (уникальный), время первого и
  последнего обращения, счётчики по типам обработанного контента (фото/изображение,
  аудио/голос, текст, команды). Учёт — через middleware aiogram (один раз на сообщение,
  независимо от хендлера), без изменения логики существующих хендлеров.
- Команда **`/stats`** — выводит: число уникальных пользователей, суммарно сообщений,
  разбивку по типам, дату первого и последнего обращения. **Только для админа.**
- Админ определяется переменной окружения `ADMIN_USER_ID` (в `.env`, не хардкодить).
  Текущий админ — Telegram id `33030141` (@Alb0203). Для остальных `/stats` либо молчит,
  либо отвечает «команда недоступна».

## Персистентность (важно для деплоя)
- Хранилище — **SQLite-файл в примонтированном volume**, чтобы данные переживали
  передеплой (контейнер пересоздаётся при каждом релизе).
- Правка `deploy/docker-compose.yml`: добавить `volumes:` (например,
  `/opt/apps/textify/data:/app/data`), путь к БД внутри — в каталоге volume.
- На сервере создать каталог данных и обеспечить право записи для непривилегированного
  пользователя контейнера (`appuser`), т.к. образ запускается не от root.
- Запись в SQLite блокирующая — не блокировать event loop (через `asyncio.to_thread`
  или аналог), как уже принято для CPU-bound операций.

## Зачем
Пользователь хочет видеть аудиторию бота и характер использования. Сейчас бот stateless и
ничего не хранит, поэтому считать нечего — учёт начнётся с момента внедрения.

## Дополнительный контекст
- План релизов: `roadmap.md`. Этот релиз занимает слот **v0.6.0**; «UX и устойчивость»
  сдвигается на v0.7.0 (обновить roadmap).
- Профиль поставки и паттерны: `adrs/adr-001.md` (long-polling, без traefik/edge/ports),
  неблокирующие вызовы (ADR-002/003/004). Релизы v0.1.0–v0.5.0 в проде на ivahub.
- Приватность: храним только Telegram `user_id` и агрегаты — без текста сообщений и
  персональных данных сверх необходимого.

## Вне рамок (не делать в этом релизе)
- Веб-дашборд, экспорт, графики.
- Хранение содержимого сообщений/распознанного текста.
- Ограничения скорости/квоты на пользователя (это UX-релиз v0.7.0).
