# req-textify-008: Транскрипция аудио через Cloudflare Whisper (v0.8.0)

## Что нужно
Перенести основную транскрипцию аудио с локальной модели faster-whisper `base` (1 vCPU,
медленно, низкое качество) на **Cloudflare Workers AI `whisper-large-v3-turbo`** —
существенно точнее и без нагрузки на CPU/RAM сервера.

Конкретно:
- `services/transcribe.py`: добавить провайдера Cloudflare. Запрос (подтверждён вживую с
  ivahub, HTTP 200):
  - `POST https://api.cloudflare.com/client/v4/accounts/{CF_ACCOUNT_ID}/ai/run/{model}`
  - заголовок `Authorization: Bearer {CF_API_TOKEN}`, тело JSON `{"audio": "<base64 аудио>"}`
  - ответ: `result.text` (+ `result.transcription_info.language`)
  - модель по умолчанию `@cf/openai/whisper-large-v3-turbo`, вынести в env `CF_WHISPER_MODEL`.
- Async через `httpx` (уже зависимость, используется в `services/structure.py`); разумный
  таймаут.
- **Переключатель провайдера** `ASR_PROVIDER` (значения `cloudflare` | `local`,
  по умолчанию `cloudflare`).
- **Фолбэк на локальный faster-whisper** при сбое CF (сетевой сбой, лимит тарифа, ошибка
  ответа) — локальная модель остаётся в образе как страховка. Сбой не должен ломать ответ
  пользователю.
- Переиспользовать уже имеющиеся `CF_ACCOUNT_ID` / `CF_API_TOKEN` (есть в `.env` на сервере
  и в `deploy/.env.example`); добавить `ASR_PROVIDER` и `CF_WHISPER_MODEL` в `.env.example`.

## Ограничения и риски
- Размер запроса CF AI run ограничен; base64 раздувает объём. Для крупных аудио —
  предусмотреть поведение: либо чанкинг (декодирование уже есть через PyAV/av в образе),
  либо фолбэк на локальную модель / сообщение, если файл слишком большой. Решение и порог —
  за архитектором (можно начать с фолбэка на local для больших файлов).
- RU IP до Cloudflare работает (структурирование уже на CF, и whisper-эндпоинт проверен —
  HTTP 200). Это снимает основной сетевой риск.
- Лимиты бесплатного тарифа Workers AI — следить; фолбэк на local смягчает.

## Зачем
Локальный `base` на 1 vCPU — главное узкое место (скорость и качество). CF
`whisper-large-v3-turbo` даёт мультиязычное распознавание заметно выше качеством, быстрее
ответ и освобождает CPU/RAM сервера. Инфраструктура CF уже интегрирована (переиспользование,
не новая зависимость).
