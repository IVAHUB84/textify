# req-textify-009: Доп-действия над результатом — «Кратко» и «Перевести» (v0.9.0)

## Что нужно
Дать пользователю быстрые действия над распознанным/структурированным текстом —
**краткое саммари** и **перевод** — по аналогии с платными ботами-конкурентами, но
бесплатно. Использовать уже работающий LLM-слой Cloudflare Workers AI.

Конкретно:
- Под текстовым результатом (OCR и аудио) показывать inline-кнопки:
  - **«Кратко»** — саммари в 3–5 пунктов на языке текста.
  - **«Перевести»** — перевод (целевой язык — решение архитектора; разумный дефолт:
    если текст русский → на английский, иначе → на русский; либо две кнопки EN/RU).
- По нажатию — вызвать CF Workers AI LLM (тот же провайдер/паттерн, что
  `services/structure.py`) с соответствующим промптом над исходным текстом и прислать
  результат отдельным сообщением (через `send_result` из v0.7.0, чтобы длинный ответ не
  обрезался).
- Источник текста для действия: текст результата, к которому прикреплены кнопки
  (`callback.message.text`) — без отдельного хранилища. Если результат был длинным и ушёл
  несколькими сообщениями/файлом (v0.7.0) — кнопки в таком случае можно не показывать
  (упрощение; решение архитектора).

## Зачем
Конкуренты (TAK! TEXT, Memo AI, Transcriber) удерживают пользователей именно доп-ценностью
(саммари, перевод, вопросы по тексту). У Textify LLM-слой уже есть — добавить эти действия
дёшево, это рост удержания (а аудиторию теперь видно через `/stats`).

## Дополнительный контекст

- LLM-интеграция: `adrs/adr-005.md` (CF Workers AI, `services/structure.py`, httpx, env

- Языки: ru+en.

## Вне рамок (не делать в этом релизе)
- Вопросы по содержимому (Q&A-чат) — отдельная идея на будущее.
- Хранение текстов между перезапусками (действия работают по тексту видимого сообщения).
- Перевод на произвольные языки сверх ru/en.
- Озвучка (TTS).
