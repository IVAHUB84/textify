# req-textify-012: Режим групп (v1.2.0)

## Что нужно
Дать боту работать в группах/супергруппах: распознавать голосовые/аудио и текст с
изображений прямо в групповых чатах. Для бота-расшифровки это главный канал органического
роста — одна установка в группу показывает бота десяткам людей.

Конкретно:
- Бот можно добавить в группу (флаг can_join_groups уже включён).
- **Не спамить.** В группе бот реагирует НЕ на каждое сообщение, а только по явному запросу,
  совместимому с privacy mode (бот по умолчанию получает только команды, упоминания и ответы
  на свои сообщения):
  - пользователь **отвечает (reply)** на голосовое/аудио/фото/документ-медиа сообщением с
    **командой** (например `/textify`) или **упоминанием** `@YourTextifyBot` → бот берёт
    `reply_to_message`, прогоняет через существующий пайплайн (OCR/ASR) и отвечает
    результатом (через `send_result`, с кнопками «Кратко»/«Перевести»).
  - фото с подписью, содержащей упоминание бота, — опционально (на усмотрение архитектора).
- В личке (private) поведение прежнее: медиа обрабатывается автоматически, без reply.
- Ответ в группе — reply на исходное сообщение, чтобы было понятно, к чему относится.

## Бесплатность под нагрузкой (важно)
- Группы могут резко поднять трафик. Защита уже есть: дневной бюджет CF (v1.1.0) и семафор
  (v1.0.0). Дополнительно рассмотреть: в группах по умолчанию использовать **локальный**
  ASR/движки (бесплатно, не тратит CF-бюджет), чтобы групповой трафик не выедал лимит,
  оставляя CF для лички. Конкретное решение/конфиг — за архитектором.
- OCR (Tesseract) и так локальный и бесплатный.

## Зачем
Цель — максимум пользователей бесплатно. Группы дают вирусную экспозицию: участники видят
имя `@YourTextifyBot` при каждом ответе. Голосовые в группах многих раздражают — авто-текст
по запросу это высокая ценность и повод добавить бота в другие чаты.
