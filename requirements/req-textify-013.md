# req-textify-013: Виральная петля — рефералы, шеринг, подпись (v1.3.0)

## Что нужно
Превратить текущих пользователей в канал привлечения новых — нативными средствами Telegram,
бесплатно для всех. Три механики:

### 1. Реферальные deep-link
- Персональная ссылка-приглашение `https://t.me/YourTextifyBot?start=ref_<user_id>`.
- В `/start` разбирать параметр `start`: если это `ref_<id>` и пользователь **новый** (первое
  обращение) и `id != сам пользователь` — зафиксировать связь «кто кого привёл».
- Хранить в существующей SQLite (рядом со статистикой v0.6.0): таблица рефералов
  (referrer_id, referred_id, время). Идемпотентно: один приглашённый учитывается один раз,
  самоприглашение игнорируется.
- В `/start` показывать пользователю его персональную ссылку и счётчик «приглашено: N».

### 2. Шеринг
- Под результатом (или в `/start`) — inline-кнопка **«Поделиться ботом»**, ведущая на
  `https://t.me/share/url?...` с ссылкой на бота (с реф-кодом приглашающего), чтобы
  пользователь в один тап делился ботом в любой чат.

### 3. Ненавязчивая подпись (attribution)
- К результату добавлять короткую строку-подпись `— @YourTextifyBot`, чтобы пересланный
  текст нёс имя бота (бесплатная реклама). Только для одно-сообщенческого результата (не
  ломать split/файл из v0.7.0). Включается/выключается env (`ATTRIBUTION_FOOTER`, дефолт on).

## Бесплатность
- Всё нативно: Telegram deep-link/share + локальная SQLite. **CF/LLM не используются** — на
  free-tier не влияет, затрат нет.

## Зачем
Цель — максимум пользователей бесплатно. Реферальные ссылки + шеринг + подпись создают
самоподдерживающуюся петлю привлечения; аудиторию и эффект видно через `/stats`
(добавить реферальные метрики).
