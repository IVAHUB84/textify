# req-textify-014: Проактивные группы — кнопка «Распознать» (v1.4.0)

## Что нужно
В группах бот должен **сам замечать** медиа и предлагать распознать его кнопкой, а не ждать
команду. Это интуитивно и виральнее (кнопка видна всем участникам).

Поведение:
- В группе/супергруппе, когда появляется **голосовое / аудио / фото / документ-изображение /
  документ-аудио**, бот отвечает (reply на это сообщение) компактной inline-кнопкой
  **«Распознать»** (для аудио — «Распознать голос», для изображений — «Распознать текст»).
- Бот **сам текст не вываливает** — расшифровка только по тапу на кнопку.
- По тапу: бот берёт исходное медиа (`callback.message.reply_to_message` — кнопка-оффер
  отправлена reply на медиа), прогоняет через существующий пайплайн (ASR/OCR), постит
  результат reply на исходное сообщение, с кнопками «Кратко»/«Перевести».
- Группы используют локальный ASR по умолчанию (`GROUP_ASR_LOCAL`, как в v1.2.0) —
  не тратит CF-бюджет лички.
- Один оффер на одно медиа-сообщение (не дублировать).

## Требует отключения privacy
- Чтобы бот видел медиа в группе, **Group Privacy должен быть ОТКЛЮЧЁН** (@BotFather
  `/setprivacy` → Disable, затем заново добавить бота в группу). Это обязательный ручной шаг
  (в чек-лист деплоя). При включённом privacy режим работать не будет.
- При privacy OFF бот видит все сообщения группы, но реагирует **только на медиа** (на
  обычный текст в группе не отвечать — не спамить).

## Совместимость
- В личке поведение прежнее (авто-распознавание без кнопок-офферов) — контракт 1.0.0
  (`adr-010`) не ломать.
- Прежний триггер `/textify`/упоминание в reply (v1.2.0) — можно сохранить как
  дополнительный путь или заменить кнопкой; решение за архитектором (приоритет — кнопка).

## Зачем
Цель — максимум пользователей бесплатно. Проактивная кнопка убирает трение (не надо знать
команду), а её появление на каждое голосовое/фото в группе показывает бота всем участникам —
сильный органический рост. Распознавание по тапу не спамит и не перегружает 1 vCPU.
