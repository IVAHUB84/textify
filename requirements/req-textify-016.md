# req-textify-016: Расширение пост-распознавательного меню действий, HTML-форматирование, персистентный кэш и устойчивость

> Требование оформлено **задним числом**: функциональность уже реализована и закоммичена
> прямыми коммитами после тега `v1.5.3` (`d322a39`…`fbbc09d`) в обход workflow. Цель документа —
> зафиксировать намерение и контракт, чтобы BA/архитектор оформили `ba-req`, `adr` и
> `release-spec` для релиза **v1.6.0**.

## Что нужно
После v1.5.0 (прогрессивное раскрытие суть → кратко → полностью) накопился набор улучшений,
взятых из второго раунда ресёрча и из практики конкурирующих ботов. Нужно оформить их единым
MINOR-релизом v1.6.0:

### 1. Telegram HTML-форматирование вывода
Сейчас структурированный текст шлётся без `parse_mode`, поэтому Markdown (`####`, `**`) виден
буквально. Конвертировать Markdown LLM в безопасное подмножество Telegram HTML (жирные
заголовки, маркированные списки, экранирование `< > &`) и слать с `parse_mode=HTML`; при
`TelegramBadRequest` — деградация в плоский текст. Промпт структурирования переучить на
Telegram-формат (жирные заголовки, списки через дефис, экономная эмодзи) вместо `#`-заголовков.

### 2. Реальное сжатие в «Кратко»
Слабая модель `llama-3.1-8b` на коротком вводе раздувала «Кратко» вместо сжатия — иногда
длиннее полного текста. Промпт `summarize` переписать на действительно более короткую выжимку
(2–5 тезисов, без деталей/повторов, никогда не длиннее источника) + few-shot на сжатие, по
аналогии с «сутью».

### 3. Расширение меню действий (три новые возможности)
- **Тайм-коды** — для длинного аудио (≥ 60 с) кнопка «Тайм-коды»: построчные `[MM:SS]` на
  основе сегментов whisper. `transcribe.py` отдаёт сегменты (`transcribe_with_timestamps`).
- **Перевести** — вернуть `translate()` (RU↔EN, язык-источник LLM определяет сам), привязать
  к кнопке «Перевести». **Это реверс удаления из v1.5.0 (ADR-015)** — нужен follow-up ADR,
  явно отменяющий соответствующее решение ADR-015.
- **Задачи** — `extract_tasks()` вытаскивает action items / решения в чек-лист за кнопкой
  «Задачи».
Перевод и задачи переиспользуют существующий кэш + CF-бюджет, идут с few-shot для слабой модели.

### 4. Субтитры SRT
Для длинного аудио — кнопка «Субтитры»: стандартный `.srt` из сегментов whisper, отправляется
документом. Тайм-коды и субтитры берут сегменты из кэша по клику, а не из заранее
сформированной строки.

### 5. Searchable PDF из фото
Под результатом фото/изображения-документа — кнопка «PDF»: пересобирает searchable PDF
(оригинал + невидимый текстовый слой Tesseract) через `pytesseract.image_to_pdf_or_hocr`; текст
остаётся выделяемым и ищущимся. Без новых зависимостей (тот же `rus+eng` Tesseract). Исходные
байты картинки — в маленьком in-memory-кэше (`IMG_CACHE_MAX_SIZE=50`, без SQLite).

### 6. Персистентный кэш результатов
`result_cache` получает опциональный SQLite-бэкенд (`init_result_cache`, write-through,
load-on-start): кнопки действий продолжают работать после редеплоя и после истечения 1h TTL,
а не падают молча в «Текст недоступен». Без инициализации — чистый in-memory (тесты,
неконфигурированные окружения).

### 7. Гард на превышение 20 МБ
Хендлеры голоса/аудио/изображений отвечают понятным сообщением, когда файл превышает лимит
Telegram getFile (20 МБ), вместо тихого падения на скачивании.

### 8. Убрать menu-кнопку команд
Перестать регистрировать `/start` и `/help` через `set_my_commands`, чтобы Telegram не
показывал кнопку Menu у поля ввода. Команды продолжают работать при ручном вводе; описания
сохранены.

## Поведение и детали
- Набор кнопок под результатом теперь шире: «Полностью», «Кратко», + контекстные «Перевести»,
  «Задачи», «Тайм-коды»/«Субтитры» (аудио), «PDF» (фото). Подсказки fallback-превью указывают на
  кнопки ниже обобщённо, без перечисления конкретных названий (`0a13011`).
- Тайм-коды/субтитры/PDF доступны только когда есть исходные данные (сегменты / байты картинки);
  по истечении кэша — понятная деградация, не падение.
- Все новые LLM-операции деградируют при исчерпании CF-бюджета (как «суть» в v1.5.0).

## Открытые вопросы для архитектора
- **Контракт вывода.** HTML-форматирование меняет формат немедленной выдачи (касается раздела
  публичного контракта 1.0.0 — ADR-010); зафиксировать как совместимое изменение представления.
- **Реверс ADR-015.** Кнопка «Перевести» возвращает удалённую функцию. Новый ADR должен явно
  пометить соответствующее решение ADR-015 как replaced/superseded, а не молча противоречить.
- **Персистентность кэша.** SQLite-бэкенд `result_cache` соседствует с учётом v0.6.0 — решить
  имя файла/таблицы, схему, TTL-чистку, поведение при отсутствии конфигурации.
- **Лимит 20 МБ** — единое сообщение и точка проверки (до скачивания) для трёх хендлеров.
- **Память на 1 vCPU/2 GB.** `IMG_CACHE` (50 элементов, только память) и SQLite-кэш не должны
  раздувать RSS; оценить.

## Зачем
- Паритет по возможностям с конкурирующими транскрипционными ботами (тайм-коды, субтитры,
  перевод, задачи, searchable PDF).
- Чистая выдача (HTML вместо сырого Markdown), честное «Кратко».
- Устойчивость: кнопки переживают редеплой; крупные файлы и истёкший кэш не роняют бота.
