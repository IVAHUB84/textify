# req-textify-018: Реферальная награда — повышение дневного лимита за приглашённых друзей

## Что нужно
Второй слой роста после subscription-gate (v1.7.0). Учёт рефералов уже работает с v1.3.0
(`services/referrals.py`: `record_referral` при `/start` с реф-ссылкой, `count_referrals`,
`top_referrers`), но число приглашённых ни на что не влияет. Нужно **связать количество
рефералов с дневным лимитом распознаваний**: пригласил друзей — получил больше распознаваний в
день. Это даёт пользователю мотив звать друзей (виральный рост) поверх подписки на канал.

### Механика награды
- За каждого подтверждённого реферала пользователь получает **+N распознаваний к дневному
  лимиту** (env `REFERRAL_BONUS_PER`, дефолт например +3), **до потолка** (env
  `REFERRAL_BONUS_CAP`, дефолт например +30), чтобы один человек не получил безлимит.
- Бонус **складывается** с базовым лимитом: эффективный дневной лимит =
  (базовый или подписчика) + min(referrals × `REFERRAL_BONUS_PER`, `REFERRAL_BONUS_CAP`).
- Реферальный бонус — **независимый рычаг** от подписки: работает и без подписки на канал.
  Подписка и рефералы дополняют друг друга.

### Где показать
- `/start`: показать пользователю его эффективный лимит и из чего он складывается (база +
  бонус за рефералов), сколько друзей он уже привёл, и его **реферальную ссылку** с призывом
  пригласить (ссылка-механизм реф-ссылки уже есть с v1.3.0).
- При исчерпании лимита: в gate-сообщении (или рядом) добавить альтернативу «или пригласи
  друзей по своей ссылке — лимit вырастет», не только «подпишись на канал».

### Опционально (на усмотрение архитектора, можно отложить)
- Уведомление пригласившему, когда по его ссылке пришёл новый пользователь («+1 друг, твой
  лимит вырос») — приятный крючок удержания. Если усложняет — вынести в отдельное требование.

## Поведение и детали
- Бонус считается по `count_referrals(user_id)` — данные уже персистентны в `stats.db`.
- Точка применения — `enforce_limit` (`handlers/gate.py`), где сейчас выбирается лимит
  subscribed/free. Туда добавляется реферальный бонус.
- Лимит остаётся дневным со сбросом в полночь UTC (как в v1.7.0). Бонус — на каждый день,
  пока рефералы числятся (постоянная награда, не разовая). Если нужно иное (разовый бонус) —
  решает архитектор, но дефолт — постоянный множитель к дневному лимиту.

## Открытые вопросы для архитектора
- **Горячий путь:** `count_referrals` — это SQL `COUNT` с индексом на каждое распознавание.
  Кэшировать (TTL, как подписку в v1.7.0) или читать напрямую? Оценить нагрузку на 1 vCPU.
- **Взаимодействие с gate-сообщением:** при исчерпании лимита показывать оба пути (подписка +
  рефералы) — как не перегрузить сообщение и клавиатуру.
- **Анти-фрод:** самоприглашение уже исключено схемой (`referred_id PRIMARY KEY`, человек не
  может пригласить сам себя через `/start`?). Мультиаккаунты — вне рамок (минимально:
  полагаться на то, что Telegram-аккаунты создавать дорого). Подтвердить, что бонус не
  накручивается тривиально.
- **Дефолты `REFERRAL_BONUS_PER` / `REFERRAL_BONUS_CAP`** — подобрать разумные, чтобы бонус
  был ощутим, но не превращал бота в безлимит.
- **Уведомление пригласившему** — включать в этот релиз или отложить.

## Совместимость / охват
- `handlers/gate.py` (`enforce_limit` — точка применения бонуса).
- `services/referrals.py` (уже есть `count_referrals` — переиспользовать; возможно добавить
  кэш-обёртку или функцию расчёта бонуса).
- `handlers/commands.py` (`/start` — показать лимит, бонус, реф-ссылку; `/stats` для админа уже
  показывает реферальные метрики с v1.3.0).
- `config.py`, `.env.example`, `deploy/.env.example` — новые переменные
  `REFERRAL_BONUS_PER`, `REFERRAL_BONUS_CAP`.
- Не ломать v1.7.0: при нуле рефералов эффективный лимit = как сейчас (база/подписчик).

## Зачем
- **Виральный рост:** прямой стимул звать друзей → больше пользователей бота → больше трафика в
  канал (через gate) → ближе к 1000+ подписчиков канала и монетизации.
- Реферальная награда дополняет subscription-gate: два независимых мотива привести пользователя
  и подписаться.
- Переиспользует уже готовую инфраструктуру рефералов (v1.3.0) — малый объём, высокий эффект.
