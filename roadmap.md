# Roadmap — Textify

Предварительный план релизов. Версионирование — semver (см. `CLAUDE.md`). До `v1.0.0`
публичный контракт считается нестабильным.

Принцип разбивки: каждый релиз — один законченный, проверяемый кусок ценности.
Деплой-контракт вводится в первом же релизе, чтобы снять риски поставки заранее.

## v0.1.0 — Каркас бота

Минимально деплоящийся бот без функций распознавания.

- aiogram, точка входа `bot.py`.
- Команды `/start`, `/help`.
- Приём сообщений, ответ-заглушка на текст.
- Конфигурация через `.env` (`BOT_TOKEN`), `.env.example` с плейсхолдерами.
- Структура каталогов (`handlers/`, `services/`) — каркас под будущие фичи.
- Деплой-контракт: `Dockerfile`, `deploy/docker-compose.yml`, `deploy/.env.example`,
  активация push-триггера в `.github/workflows/deploy.yml` (первый релиз продукта).

Критерий приёмки: бот запускается локально и на VPS, отвечает на `/start`.

## v0.2.0 — Картинки → текст (OCR)

- `handlers/image.py` — приём фото/изображений.
- `services/ocr.py` — обёртка над `pytesseract`, языки `rus` + `eng`.
- Выдача распознанного сырого текста без постобработки.

Системная зависимость: `tesseract-ocr`, `tesseract-ocr-rus`, `tesseract-ocr-eng`.

Критерий приёмки: на фото с русским и английским текстом бот возвращает распознанный текст.

## v0.3.0 — Аудио → текст (faster-whisper)

- `handlers/audio.py` — приём голосовых и аудиофайлов.
- `services/transcribe.py` — обёртка над faster-whisper, автоопределение языка.
- Декодирование `.oga`/`.ogg`/прочих форматов через PyAV (`av`), системный `ffmpeg` не добавляется.
- Веса модели вшиты в Docker-образ на этапе сборки; в рантайме сетевой доступ к HuggingFace не нужен.
- Выдача сырого транскрипта без постобработки.

Критерий приёмки: на голосовом сообщении (rus/eng) бот возвращает транскрипт.

## v0.4.0 — Улучшение качества OCR (предобработка изображений)

- `services/preprocess.py` — конвейер предобработки изображений на OpenCV/PIL/numpy:
  EXIF-ориентация, grayscale, кэп размера (2500 px), апскейл мелких (→ 1500 px),
  лёгкий деноизинг (`medianBlur`), адаптивная бинаризация, deskew (±15°).
- `services/ocr.py` — интеграция предобработки; настройки Tesseract `--oem 1 --psm 3`;
  фолбэк на исходное изображение при сбое предобработки.
- UX-подсказка слать изображение как файл (документ) в `/help` и в сообщении о нераспознанном тексте.
- Зависимости: `opencv-python-headless`, `numpy`; системные библиотеки `libglib2.0-0`, `libgomp1`.

Примечание: Groq-структурирование перенесено на v0.5.0 — API недоступен с российских IP
(гео-блокировка Groq по RU IP; недоступно с ivahub).

Критерий приёмки: заметный прирост точности OCR на типовых пользовательских фото;
контракт `recognize_text` не меняется; фолбэк работает.

## v0.5.0 — Структурирование текста (Cloudflare Workers AI; Groq — опция)

Groq гео-блокирует российские IP (CF 403): с ivahub API недоступен. Провайдер по умолчанию
заменён на Cloudflare Workers AI — работает с RU-IP без прокси. Groq сохранён переключаемой
опцией (`LLM_PROVIDER=groq`) на случай решения проблемы доступности.

- `services/structure.py` — оркестратор структурирования с абстракцией провайдера.
- Провайдер по умолчанию — Cloudflare Workers AI (`@cf/meta/llama-3.1-8b-instruct`),
  переключается через `LLM_PROVIDER`; модель переопределяется `CF_MODEL`.
- Groq — опциональный провайдер (`LLM_PROVIDER=groq`, `GROQ_API_KEY`).
- Сырой текст из OCR и транскрипции приводится к читаемой структуре Markdown:
  заголовки, списки, ключевые пункты.
- Подключается к обоим входным каналам (картинки и аудио).
- При любом сбое LLM (сеть, таймаут, ошибка, отсутствие ключей) — тихий фолбэк на сырой текст.
- Новые переменные окружения: `LLM_PROVIDER`, `CF_ACCOUNT_ID`, `CF_API_TOKEN`, `CF_MODEL`,
  `GROQ_API_KEY` — все опциональны (бот стартует без них, деградирует до фолбэка).

Критерий приёмки: результат любого канала возвращается структурированным (или сырым при недоступности LLM).

## v0.6.0 — Учёт пользователей и команда /stats

Выпущен 2026-06-22.

- Сквозной учёт обращений через outer-middleware aiogram (`middlewares/stats.py`) — без правки существующих хендлеров.
- Персистентное хранилище: SQLite с WAL в bind-mount volume (`/opt/apps/textify/data:/app/data`).
- Команда `/stats` для администратора: уникальные пользователи, всего сообщений, разбивка по типам (фото, аудио, текст, команды, прочее), даты первого и последнего обращения (UTC).
- Авторизация через `ADMIN_USER_ID` из окружения; не-админу — короткий отказ без раскрытия данных.
- Тихая деградация: сбой учёта не прерывает основную функцию бота.
- Первое персистентное состояние — осознанное расширение профиля поставки ADR-001 (ADR-006).

Критерий приёмки: владелец бота видит сводку `/stats`; учёт переживает передеплой.

## v0.7.0 — UX и устойчивость

**Закрыто в этом релизе:**

- Надёжная отдача длинного результата: единая точка отдачи (`services/reply.py`), разбивка по
  естественным границам (абзацы/строки/предложения), переход на файл `.txt` при очень длинном
  результате, plain-text без `parse_mode`. Применяется ко всем путям вывода (OCR, аудио).

**Остаётся в теме «UX и устойчивость» — следующие итерации:**

- Индикация прогресса обработки (typing/статус-сообщения).
- Понятные пользовательские сообщения при таймаутах/ошибках (сейчас тихий фолбэк на сырой текст).
- Лимиты размера входного файла, валидация форматов.

Критерий приёмки: бот корректно обрабатывает граничные случаи без падений.

## v0.8.0 — Транскрипция аудио через Cloudflare Whisper

Основная транскрипция аудио переведена на Cloudflare Workers AI `whisper-large-v3-turbo` (точнее,
мультиязычнее, быстрее, снимает CPU/RAM-нагрузку с ivahub). Локальный faster-whisper сохраняется
как фолбэк на любой сбой облака и как самостоятельный режим (`ASR_PROVIDER=local`).

- Облачный провайдер Cloudflare Workers AI `whisper-large-v3-turbo` — дефолт; async httpx без
  блокировки event loop (нативный `await`, без `to_thread`).
- Локальный faster-whisper — фолбэк на любой сбой облака (сеть, таймаут, лимит, ошибка ответа) и
  явный режим `ASR_PROVIDER=local`; факт фолбэка логируется.
- Переключатель `ASR_PROVIDER` (дефолт `cloudflare`); отдельная переменная модели `CF_WHISPER_MODEL`
  (дефолт `@cf/openai/whisper-large-v3-turbo`), независимая от `CF_MODEL` структурирования.
- Порог 8 МБ для крупных файлов: превышение → локальный путь без обращения к Cloudflare.
- Единичная попытка timeout=60, без ретраев.
- Новых обязательных секретов нет; `CF_ACCOUNT_ID` / `CF_API_TOKEN` переиспользуются (ADR-005).
- Профиль поставки (ADR-001), путь вывода (ADR-007) и контракт канала аудио не меняются.

Критерий приёмки: аудио распознаётся через Cloudflare Whisper; при сбое облака срабатывает
локальный фолбэк без ошибки пользователю.

## v0.9.0 — Доп-действия над результатом

- Под одно-сообщенческим результатом OCR и аудио отображаются inline-кнопки **«Кратко»** и **«Перевести»**.
- `services/llm.py` — саммари в 3–5 пунктов на языке исходника и перевод ru↔en через CF Workers AI.
- Эвристика направления перевода: наличие кириллицы → на английский, иначе → на русский.
- `handlers/actions.py` — роутер с callback-хендлерами; кнопки не показываются под длинным результатом (серия/файл).
- Профиль поставки (ADR-001), env, зависимости без изменений.

Критерий приёмки: пользователь нажимает «Кратко»/«Перевести» под текстовым результатом и получает ответ; при сбое CF — понятное сообщение, бот не падает.

## v1.0.0 — Продакшн-готовность (выпущен 2026-06-23)

- Наблюдаемость: `logging.basicConfig` с уровнем из `LOG_LEVEL`, `PYTHONUNBUFFERED=1` в Dockerfile.
- Индикатор прогресса: `ChatActionSender` в хендлерах изображений, аудио и доп-действий.
- Защита от перегрузки: глобальный `asyncio.Semaphore(1)` на тяжёлые локальные операции (OCR, локальный Whisper).
- Полный рерайт `README.md` под фактическое состояние с разделом эксплуатации и полным списком env.
- Фиксация публичного контракта 1.0.0 (команды, входные каналы, формат ответа, inline-действия).

Критерий приёмки: продукт стабилен на проде, контракт зафиксирован.

## v1.1.0 — Гарантия бесплатности при росте + дискаверабилити (выпущен 2026-06-23)

- Дневной счётчик обращений к Cloudflare Workers AI (`cf_usage` в SQLite, тот же volume).
- Авто-деградация при исчерпании бюджета: ASR → локальный faster-whisper; структурирование → сырой
  текст; действия «Кратко»/«Перевести» → понятное сообщение пользователю.
- Опциональная env `CF_DAILY_BUDGET` (дефолт 300); некорректное значение нормализуется к дефолту.
- Усиление ключевых слов в метаданных бота (транскрипция, OCR, перевод, RU/EN) под поиск Telegram.
- Готовый листинг для каталогов ботов (`docs/listing.md`).

Критерий: рост трафика не приводит к оплате Cloudflare; пользователь не получает отказа.

## v1.2.0 — Режим групп (выпущен 2026-06-23)

Поддержка использования бота в групповых чатах Telegram.

- Бот обрабатывает медиа в группах/супергруппах по явному запросу: reply на медиа + `/textify` или упоминание `@bot`.
- Результат отдаётся reply на исходное медиа через `send_result` с кнопками «Кратко»/«Перевести».
- ASR в группах по умолчанию форсируется локальным faster-whisper (`GROUP_ASR_LOCAL=true`), не расходуя CF-бюджет лички.
- Поведение в личке не изменено (публичный контракт 1.0.0).

## v1.3.0 — Виральная петля: рефералы, шеринг, подпись (выпущен 2026-06-23)

Завершает текущий план роста. Нативные бесплатные механики без новых зависимостей и портов.

- Реферальные deep-link `?start=ref_<id>`: фиксация связи «кто кого привёл» (таблица `referrals` в существующей SQLite, `INSERT OR IGNORE`, идемпотентность на уровне БД).
- Счётчик «Приглашено: N» и персональная реф-ссылка в `/start`.
- Inline-кнопка «Поделиться ботом» на `t.me/share/url` с персональной ссылкой.
- Ненавязчивая подпись `— @<bot>` под одно-сообщенческим результатом распознавания (управляется env `ATTRIBUTION_FOOTER`, дефолт on).
- Реферальные метрики в `/stats` (только для админа): всего рефералов, топ-5 приглашающих.
- Признак «новый пользователь» определяется атомарно в `record_message` (UPSERT), пробрасывается через middleware в `/start` — реферал фиксируется корректно несмотря на outer-middleware-учёт.

## v1.4.0 — Проактивные группы (кнопка Распознать) (выпущен 2026-06-23)

Группы стали проактивными: бот сам замечает появившееся медиа и предлагает распознать его кнопкой — без необходимости знать команду.

- При появлении голосового/аудио/фото/документа в группе бот отправляет reply на медиа с inline-кнопкой «Распознать голос» или «Распознать текст».
- Распознавание происходит только по тапу кнопки (`callback_data = "grec"`); в момент оффера текст не отправляется.
- На обычный текст и нерелевантный трафик бот не реагирует.
- Прежний триггер v1.2.0 (reply + `/textify`/упоминание) сохранён как дополнительный путь; оба пути ведут в общий `_dispatch_media`.
- ASR в группах форсируется локальным faster-whisper (`GROUP_ASR_LOCAL`, дефолт on) — CF-бюджет лички не расходуется.
- **Обязательный ручной шаг деплоя:** @BotFather → `/setprivacy` → Disable → заново добавить бота в группу. Без этого офферы не появляются, бот не падает.

## v1.5.0 — Прогрессивное раскрытие результата (суть → кратко → полностью)

- В личке на распознанном медиа первым приходит суть (1–2 предложения) с двумя кнопками: «Показать полностью» и «Кратко». Полный текст — по кнопке, не сразу.
- In-memory LRU+TTL кэш `message_id → исходный текст` для callback'ов (заменяет чтение `callback.message.text`).
- Новая LLM-операция `summarize_gist` — суть на языке оригинала; деградация при недоступности CF-бюджета → служебное превью + рабочие кнопки.
- Группы сохраняют прежнюю немедленную выдачу; клавиатура — одна кнопка «Кратко» (без «Перевести»).
- Кнопка «Перевести» и функция `translate` удалены полностью.
- Ленивый расчёт «Полностью»/«Кратко» по тапу частично компенсирует добавленный CF-вызов «сути».

## v1.6.0 — Расширенное меню действий, HTML-вывод, персистентный кэш (выпущен 2026-06-24)

- **HTML-форматирование:** структурированный текст отправляется с `parse_mode=HTML` — жирные заголовки, маркированные списки (`•`), экранирование `< > &`; деградация в плоский текст при ошибке разметки Telegram.
- **Честное «Кратко»:** промпт `summarize` переписан на выжимку 2–5 тезисов с few-shot сжатия; результат всегда короче исходного.
- **Возврат «Перевести»** (реверс ADR-015): `translate(RU↔EN)` через CF Workers AI, направление определяет LLM.
- **Новые LLM-действия:** «Задачи» (`extract_tasks`) — action items / решения в чек-лист.
- **Тайм-коды и субтитры:** для аудио ≥ 60 с — кнопки «⏱ Тайм-коды» (текст `[MM:SS]`) и «Субтитры» (файл `.srt`) из сегментов whisper; без LLM-вызова.
- **Searchable PDF:** для изображений — кнопка «📄 PDF»; PDF = исходное изображение + невидимый текстовый слой Tesseract `rus+eng`.
- **Персистентный кэш результатов:** SQLite write-through `results.db` рядом со `stats.db`; кнопки действий выживают редеплой и истечение TTL; без инициализации — чистый in-memory (поведение v1.5.0).
- **Гард 20 МБ:** единая точка проверки `is_oversized` в `handlers/limits.py`; понятное сообщение до попытки скачивания.
- **Убрана кнопка Menu:** `setup_bot_profile` вызывает `delete_my_commands`; `/start`/`/help` работают при ручном вводе.

## v1.7.0 — Дневной лимит распознаваний + subscription-gate

- Дневной счётчик распознаваний в SQLite (`daily_usage` в существующем `stats.db`); атомарный UPSERT под WAL, ключ суток — календарный день UTC.
- Два уровня лимита: базовый `DAILY_LIMIT_FREE` (3) и подписчика `DAILY_LIMIT_SUBSCRIBED` (30) — из env.
- Мягкий subscription-gate: при исчерпании базового лимита в личке — предложение подписаться на канал `REQUIRED_CHANNEL` с кнопками «Открыть канал» / «Я подписался / Проверить». В группах — нейтральное сообщение без кнопок.
- Кэш подтверждённой подписки: in-memory TTL 600 с; `getChatMember` вызывается только по кнопке «Проверить».
- Деградация: сбой `getChatMember` → базовый лимит без gate, лог (бот не падает).
- При незаданном `REQUIRED_CHANNEL` gate выключен — dev/CI/текущий прод не ломаются.
- Кнопки-действия над результатом (ADR-016) лимит не расходуют.
- Остаток лимита в `/start` (личка), агрегат в `/stats` (админ).
- **Деплой-предусловие:** при заданном `REQUIRED_CHANNEL` бот должен быть администратором канала.

## v1.8.0 — Реферальная награда

- Эффективный дневной лимит = `база (v1.7.0) + min(друзья × REFERRAL_BONUS_PER, REFERRAL_BONUS_CAP)`.
- Бонус начисляется независимо от подписки и складывается с уровнем подписчика.
- При нуле рефералов поведение идентично v1.7.0 (без регрессии).
- При исчерпании эффективного лимита в личке неподписчик видит два пути: подписка на канал и приглашение друзей (кнопки «Открыть канал», «Пригласить друзей», «Проверить»).
- `/start` показывает эффективный лимит; при наличии бонуса — расшифровку «база + бонус за N друзей».
- TTL-кэш числа рефералов (600 с) на горячем пути; точечная инвалидация при фиксации нового реферала.
- Новые env: `REFERRAL_BONUS_PER` (3), `REFERRAL_BONUS_CAP` (30).
